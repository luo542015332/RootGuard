# PandaSU 超级用户列表加载逻辑

## 概述

PandaSU 的超级用户列表功能是从 Root 系统（Magisk/KernelSU）中读取应用的 Root 权限策略，并结合设备上所有已安装应用，生成一个完整的应用权限列表。

## 数据流程

### 1. 主要入口函数

**位置**: `app/src/main/java/com/rootguard/app/data/repository/MagiskRepository.kt`

```kotlin
suspend fun getAppsWithRootStatus(): List<AppItem>
```

### 2. 数据获取步骤

#### 步骤 1: 获取所有已安装应用
```kotlin
val allApps = magiskProvider.getAllInstalledApps()
```

**位置**: `app/src/main/java/com/rootguard/app/data/magisk/MagiskProvider.kt`  
**函数**: `suspend fun getAllInstalledApps(): List<InstalledAppInfo>`

**获取方式（按优先级）**:

1. **KernelSU 优先**: 使用 `ksud app-list` 命令
   ```bash
   su -c ksud app-list
   ```
   
2. **备选方案**: 使用 `pm list packages` 命令
   ```bash
   su -c pm list packages
   ```
   解析输出，获取所有包名，然后通过 `PackageManager.getApplicationInfo()` 获取应用信息

3. **最后回退**: 使用 PackageManager API
   ```kotlin
   pm.getInstalledApplications(flags)
   ```

#### 步骤 2: 获取 Root 权限策略
```kotlin
val policies = magiskProvider.getAppPolicies()
val policyMap = policies.associateBy { it.packageName }
```

**位置**: `app/src/main/java/com/rootguard/app/data/magisk/MagiskProvider.kt`  
**函数**: `suspend fun getAppPolicies(): List<AppPolicy>`

**获取方式**:

##### Magisk 数据库
- **数据库路径**: `/data/adb/magisk.db`
- **表名**: `policies`
- **字段**: `package_name`, `policy`, `logging`, `notification`
- **查询命令**:
  ```bash
  su -c sqlite3 /data/adb/magisk.db "SELECT package_name, policy, logging, notification FROM policies"
  ```

##### KernelSU 数据库
**支持的 KernelSU 管理器**:
1. **Weishu KernelSU** (`me.weishu.kernelsu`)
   - 数据库路径: `/data/data/me.weishu.kernelsu/databases/app.db`
   - 表名: `rules` 或 `uid_policy`
   - 字段: `uid`, `policy`

2. **Tiann KernelSU** (`cn.tianshaokuo.top.kernelsu`)
   - 数据库路径: `/data/data/cn.tianshaokuo.top.kernelsu/databases/kernelsu.db`
   - 表名: `uid_policy`
   - 字段: `uid`, `policy`

**查询流程**:
1. 检测使用哪个 KernelSU 管理器
2. 检查数据库文件是否存在
3. 检查表名 (`rules` 或 `uid_policy`)
4. 执行查询获取 uid 和 policy
5. 通过 `PackageManager.getPackagesForUid(uid)` 转换为包名

**查询命令**:
```bash
su -c sqlite3 <db_path> "SELECT uid, policy FROM <table_name>"
```

#### 步骤 3: 映射 Root 权限状态
```kotlin
val status = when {
    isKernelSU -> {
        // KernelSU: 0=拒绝, 1=允许
        when (policy?.policy) {
            1 -> RootAccessStatus.GRANTED
            0 -> RootAccessStatus.DENIED
            else -> RootAccessStatus.PROMPT
        }
    }
    else -> {
        // Magisk: 0=询问, 1=拒绝, 2=允许
        when (policy?.policy) {
            2 -> RootAccessStatus.GRANTED
            1 -> RootAccessStatus.DENIED
            else -> RootAccessStatus.PROMPT
        }
    }
}
```

#### 步骤 4: 生成应用列表
```kotlin
AppItem(
    packageName = appInfo.packageName,
    name = appInfo.appName,
    rootStatus = status,  // GRANTED, DENIED, PROMPT
    isSystemApp = appInfo.isSystemApp,
    icon = appInfo.icon,
    isIsolated = false,
    isolationLevel = "未设置"
)
```

#### 步骤 5: 排序
```kotlin
.sortedWith(
    compareByDescending<AppItem> { it.rootStatus == RootAccessStatus.GRANTED }
        .thenByDescending { it.isIsolated }
        .thenBy { it.name }
)
```

排序规则:
1. 已授权的应用优先
2. 已设置隔离的应用优先
3. 按应用名称排序

## 应用类型判断

**位置**: `app/src/main/java/com/rootguard/app/data/magisk/MagiskProvider.kt`  
**函数**: `private fun isSystemApp(appInfo: ApplicationInfo): Boolean`

**判断逻辑（修复 HyperOS/MIUI 兼容性）**:

```kotlin
private fun isSystemApp(appInfo: android.content.pm.ApplicationInfo): Boolean {
    val sourceDir = appInfo.sourceDir ?: return false
    
    // 用户应用的标准路径
    if (sourceDir.startsWith("/data/app/")) {
        return false
    }
    
    // 系统应用安装在 /system/、/vendor/、/product/、/apex/ 等目录下
    val isSystemPath = sourceDir.startsWith("/system/") ||
                      sourceDir.startsWith("/vendor/") ||
                      sourceDir.startsWith("/product/") ||
                      sourceDir.startsWith("/apex/")
    
    return isSystemPath
}
```

**注意**: 
- 不使用 `FLAG_SYSTEM` 标志，因为 HyperOS/MIUI 会错误设置
- 优先使用安装路径判断，更加可靠

## Root 类型检测

**函数**: `detectRootType()`

**检测流程**:
1. 检查 Magisk 数据库是否存在
2. 检查 KernelSU 管理器是否安装
3. 返回 Root 类型 (`ROOT_TYPE_MAGISK` 或 `ROOT_TYPE_KERNELSU`)

## 数据模型

### AppPolicy
```kotlin
data class AppPolicy(
    val packageName: String,
    val policy: Int,        // Magisk: 0=询问, 1=拒绝, 2=允许 | KernelSU: 0=拒绝, 1=允许
    val logging: Int,
    val notification: Int
)
```

### AppItem
```kotlin
data class AppItem(
    val packageName: String,
    val name: String,
    val rootStatus: RootAccessStatus,  // GRANTED, DENIED, PROMPT
    val isSystemApp: Boolean,
    val icon: Drawable?,
    val isIsolated: Boolean,
    val isolationLevel: String
)
```

## 权限设置

**函数**: `setAppRootAccess(packageName: String, status: RootAccessStatus): Boolean`

**流程**:
1. 检测 Root 类型
2. 根据 Root 类型转换 status 为 policy 值
3. 调用 `magiskProvider.setAppPolicy(packageName, policy)`

### Magisk Policy 值
- 0 = 询问 (PROMPT)
- 1 = 拒绝 (DENIED)
- 2 = 允许 (GRANTED)

### KernelSU Policy 值
- 0 = 拒绝 (DENIED)
- 1 = 允许 (GRANTED)
- 不支持询问（默认为允许）

## 性能优化

1. **延迟加载图标**: 在 `getAllInstalledApps()` 中不立即加载图标，提高首次加载速度
2. **使用命令获取应用列表**: 优先使用 `ksud app-list` 或 `pm list packages`，比 PackageManager API 更快
3. **缓存策略**: 通过 Repository 层缓存应用列表

## 已知问题与修复

### HyperOS/MIUI 应用分类问题
**问题**: HyperOS/MIUI 滥用 `FLAG_SYSTEM` 标志，导致用户应用被错误识别为系统应用  
**修复**: 改用安装路径判断，不再依赖 `FLAG_SYSTEM` 标志  
**影响版本**: v1.4.37+
