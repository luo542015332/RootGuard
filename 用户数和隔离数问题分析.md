# 用户数和隔离数不对问题分析

## 问题描述
用户反馈"用户数和隔离数还是不对"，需要分析应用列表显示的统计数据。

### 3. KernelSU 的隔离机制

**当前实现**: RootHider.kt 尝试使用 Magisk 的 `magisk --denylist` 命令

```kotlin
val process = Runtime.getRuntime().exec(
    "su -c magisk --denylist add ${config.packageName}"
)
```

**问题**: 这个命令在 KernelSU 环境下可能不工作，因为：
1. KernelSU 不使用 Magisk 的 DenyList
2. KernelSU 使用自己的隔离机制（`ksu denylist` 或 `/data/adb/ksu/denylist`）

## 可能的问题原因

### 1. 用户数和系统数不准确

#### 原因A: 系统应用判断错误
某些用户应用被错误地标记为系统应用，因为：
- MIUI/自定义 ROM 可能将某些用户应用标记为 FLAG_SYSTEM
- 从应用商店安装的应用有时会被标记为系统应用

#### 原因B: 应用加载失败
PackageManager 返回了应用列表，但某些应用处理失败，导致它们没有被添加到列表中：
```kotlin
// MagiskProvider.kt
val apps = packageManager.getInstalledApplications(PackageManager.GET_META_DATA)
// 如果某些应用处理失败，可能会跳过
```

### 2. 隔离数为空或不准确

#### 原因A: 隔离配置未正确应用
- RootHider 使用 `magisk --denylist` 命令，但在 KernelSU 环境下不工作
- 隔离配置保存在 DataStore 中，但实际系统层面没有生效

#### 原因B: 隔离状态不同步
- DataStore 中的 `isEnabled` 标记与实际系统状态不同步
- 应用重启后，系统层面的隔离配置可能被重置

## KernelSU 隔离机制调研

### KernelSU DenyList

KernelSU 提供了类似于 Magisk 的 DenyList 功能，但命令不同：

```bash
# KernelSU DenyList 命令
ksu denylist list                          # 列出所有被隔离的应用
ksu denylist add <package_name>           # 添加应用到隔离列表
ksu denylist rm <package_name>            # 移除应用
ksu denylist clear                         # 清空隔离列表
```

### KernelSU 配置文件

KernelSU 的隔离配置存储在：
```
/data/adb/ksu/denylist
```

文件格式（每行一个包名）：
```
com.example.app1
com.example.app2
```

### KernelSU 版本信息获取

```bash
ksu version           # 显示 KernelSU 版本
ksu info              # 显示详细信息
```

## 解决方案

### 方案1: 改进系统应用判断

**目标**: 更准确地判断用户应用和系统应用

**实现** (`MagiskProvider.kt`):
```kotlin
private fun isSystemApp(appInfo: ApplicationInfo): Boolean {
    // 方法1: 检查 FLAG_SYSTEM 标志
    if ((appInfo.flags and ApplicationInfo.FLAG_SYSTEM) != 0) {
        // 进一步检查安装路径，排除被错误标记的用户应用
        val isUserInstalled = appInfo.sourceDir?.contains("/data/app/") == true
        return !isUserInstalled
    }
    return false
}
```

### 方案2: 添加 KernelSU 支持

**目标**: 在 RootHider 中添加 KernelSU 特定的隔离机制

**实现** (`RootHider.kt`):
```kotlin
private suspend fun configureKernelSUDenyList(packageName: String): Boolean = withContext(Dispatchers.IO) {
    try {
        // 检测是否为 KernelSU
        val isKernelSU = checkKernelSUInstalled()
        
        if (isKernelSU) {
            // 使用 KernelSU 的 denylist 命令
            val process = Runtime.getRuntime().exec(
                "su -c ksu denylist add $packageName"
            )
            process.waitFor()
            return@withContext process.exitValue() == 0
        } else {
            // 使用 Magisk 的 denylist 命令
            val process = Runtime.getRuntime().exec(
                "su -c magisk --denylist add $packageName"
            )
            process.waitFor()
            return@withContext process.exitValue() == 0
        }
    } catch (e: Exception) {
        Logger.e("Failed to configure denylist", e)
        false
    }
}

private fun checkKernelSUInstalled(): Boolean {
    return try {
        val process = Runtime.getRuntime().exec("which ksu")
        process.waitFor()
        process.exitValue() == 0
    } catch (e: Exception) {
        false
    }
}
```

### 方案3: 添加同步机制

**目标**: 确保隔离配置与系统状态同步

**实现** (`IsolationViewModel.kt`):
```kotlin
/**
 * 同步隔离配置与系统状态
 */
fun syncIsolationWithSystem() {
    viewModelScope.launch {
        try {
            // 获取系统中的隔离应用列表
            val systemDeniedApps = getSystemDenyList()
            
            // 获取本地配置
            val localConfigs = isolationDataStore.isolationConfigs.first()
            
            // 同步差异
            localConfigs.forEach { config ->
                val isSystemDenied = systemDeniedApps.contains(config.packageName)
                if (config.isEnabled != isSystemDenied) {
                    // 同步本地配置
                    val updatedConfig = config.copy(isEnabled = isSystemDenied)
                    isolationDataStore.saveIsolationConfig(updatedConfig)
                }
            }
            
            successMessage = "隔离配置已同步"
        } catch (e: Exception) {
            errorMessage = "同步失败: ${e.message}"
        }
    }
}

private suspend fun getSystemDenyList(): Set<String> = withContext(Dispatchers.IO) {
    try {
        // 检测是 KernelSU 还是 Magisk
        val isKernelSU = checkKernelSUInstalled()
        
        val command = if (isKernelSU) {
            "su -c ksu denylist list"
        } else {
            "su -c magisk --denylist ls"
        }
        
        val process = Runtime.getRuntime().exec(command)
        process.waitFor()
        
        val output = process.inputStream.bufferedReader().readText()
        output.lines().filter { it.isNotBlank() }.toSet()
    } catch (e: Exception) {
        emptySet()
    }
}
```

### 方案4: 添加详细日志

**目标**: 在应用加载过程中记录详细信息，便于调试

**实现** (`MagiskProvider.kt`):
```kotlin
fun getAllInstalledApps(): List<AppInfo> {
    try {
        val apps = packageManager.getInstalledApplications(PackageManager.GET_META_DATA)
        
        Logger.d("PackageManager returned ${apps.size} applications")
        
        val result = mutableListOf<AppInfo>()
        val failedApps = mutableListOf<String>()
        
        apps.forEach { appInfo ->
            try {
                val app = convertToAppInfo(appInfo)
                if (app != null) {
                    result.add(app)
                } else {
                    failedApps.add(appInfo.packageName)
                }
            } catch (e: Exception) {
                Logger.e("Failed to process app: ${appInfo.packageName}", e)
                failedApps.add(appInfo.packageName)
            }
        }
        
        val userApps = result.filter { !it.isSystemApp }
        val systemApps = result.filter { it.isSystemApp }
        
        Logger.d("User apps: ${userApps.size}")
        Logger.d("System apps: ${systemApps.size}")
        Logger.d("Failed to process apps: ${failedApps.size}")
        
        if (failedApps.isNotEmpty()) {
            Logger.w("Failed apps: ${failedApps.joinToString(", ")}")
        }
        
        return result
    } catch (e: Exception) {
        Logger.e("Failed to get installed apps", e)
        return emptyList()
    }
}
```
