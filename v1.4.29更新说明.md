# v1.4.29 更新说明

## 问题分析

从日志中发现 MIUI 系统的特殊行为导致系统应用判断错误：

```
App[12]: com.milink.service | sourceDir: /data/app/... | flags: 818429637 | isSystem: true
```

**问题原因：**
- `com.milink.service` 安装在 `/data/app/`（用户应用标准路径）
- 但 MIUI 给它设置了 `FLAG_SYSTEM` 标志
- 原代码优先检查 `FLAG_SYSTEM` 标志，导致误判为系统应用

## 修复方案

修改 `isSystemApp()` 函数，**优先使用安装路径判断**：

```kotlin
private fun isSystemApp(appInfo: android.content.pm.ApplicationInfo): Boolean {
    // 方法1：优先检查安装路径（更准确）
    val sourceDir = appInfo.sourceDir ?: return false

    // 系统应用路径
    val isSystemPath = sourceDir.startsWith("/system/") ||
                      sourceDir.startsWith("/system_ext/") ||
                      sourceDir.startsWith("/vendor/") ||
                      sourceDir.startsWith("/product/") ||
                      sourceDir.startsWith("/apex/")

    if (isSystemPath) {
        return true
    }

    // 方法2：检查是否在用户目录
    val isUserPath = sourceDir.startsWith("/data/app/")
    if (isUserPath) {
        return false  // 忽略 FLAG_SYSTEM 标志
    }

    // 方法3：其他情况下才检查 FLAG_SYSTEM
    return (appInfo.flags and ApplicationInfo.FLAG_SYSTEM) != 0
}
```

**修复逻辑：**
1. 优先使用 `sourceDir` 路径判断（更可靠）
2. `/data/app/` 下的应用一律视为用户应用，忽略 `FLAG_SYSTEM`
3. 其他路径才回退到 `FLAG_SYSTEM` 标志判断

## 预期效果

修复后，用户应用数量应该从 4 恢复到正常水平（100+），包括：
- 微信（com.tencent.mm）
