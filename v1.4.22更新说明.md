# PandaSU v1.4.22 更新说明

## 🐛 发现的问题

### v1.4.21 的问题
```
pm list packages returned 133 packages
Using PackageManager API
Final packages count: 302
```

**问题分析**：
- `pm list packages -3` 只返回了 133 个包
- `-3` 参数的意思是 **只返回第三方用户应用**
- 但 133 < 302（PackageManager API 返回的数量）
- 智能降级逻辑判定 `pm 命令` 返回更少，所以选择了 PackageManager API
- 结果还是只显示 302 个应用，4 个用户应用

## 🔧 修复方案

### 移除 -3 参数
```kotlin
// v1.4.21（错误）
pm list packages -3  // 只返回第三方应用（133 个）

// v1.4.22（正确）
pm list packages  // 返回所有应用（用户 + 系统，约 492 个）
```

### pm 命令参数说明
| 参数 | 作用 | 返回数量 |
|-----|------|---------|
| 无参数 | 列出所有应用 | ~492 |
| `-3` | 只列出第三方用户应用 | 133 |
| `-s` | 只列出系统应用 | ~359 |
| `--show-versioncode` | 显示版本号 | - |

### 智能降级逻辑（保持不变）
```kotlin
val usePmCommand = packageNames.size > 302
// 如果 pm 命令返回超过 PackageManager API 的数量，使用 pm 命令结果
// 否则回退到 PackageManager API
```
