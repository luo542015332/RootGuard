# PandaSU v1.4.21 更新说明

## 🎯 核心修复
使用 `pm` 命令直接获取完整的应用列表，**绕过 PackageManager API 的限制**。

## 问题描述

### PackageManager API 的根本限制
经过多个版本的尝试（v1.4.12-v1.4.20），发现：
- `ApplicationInfoFlags.of(0)` → 只返回 301 个应用
- `MATCH_ALL` → 只返回 301 个应用
- 组合 4 个 flags → 只返回 302 个应用（只多 1 个）

**结论**：PackageManager API 在某些情况下（可能是 Android 13 的某个特定版本、厂商定制 ROM 等）确实无法获取所有应用。

### 实际情况
- 系统实际安装了 **492 个应用**（通过 `pm list packages` 验证）
- 但 PackageManager API 只返回 **302 个应用**
- 微信、QQ 等关键应用不在 PackageManager 返回的列表中

## 新的解决方案

### 使用 ADB 命令获取完整列表
```kotlin
val packageNames = try {
    val process = Runtime.getRuntime().exec(
        arrayOf("su", "-c", "pm", "list", "packages", "-3")
    )
    val output = process.inputStream.bufferedReader().readText()
    process.waitFor()
    output.lines()
        .map { it.substringAfter("package:").trim() }
        .filter { it.isNotEmpty() }
} catch (e: Exception) {
    Logger.w("Failed to get packages via pm command: ${e.message}")
    emptyList()
}
```

### pm 命令参数说明
- `pm list packages` - 列出所有包
- `-3` - 只列出第三方应用（用户应用）
- `-s` - 只列出系统应用
- 无参数 - 列出所有应用

### 智能降级策略
```kotlin
val usePmCommand = packageNames.size > 302
// 如果 pm 命令返回超过 PackageManager API 的数量，使用 pm 命令结果
// 否则回退到 PackageManager API（原方案）
```

## 技术细节

### 为什么 pm 命令能获取更多应用？
`pm list packages` 是 Android 的 shell 命令，直接访问包管理器的底层数据库，不受 PackageManager API 的限制。

### 智能切换逻辑
```kotlin
// 比较 pm 命令和 PackageManager API 的结果
if (pmCommandResult.size > packageManagerResult.size) {
    // pm 命令返回更多应用，使用 pm 命令结果
    usePmCommand = true
} else {
    // PackageManager API 返回更多或相同，使用 API（更快）
    usePmCommand = false
}
```

### 包名列表处理
```kotlin
output.lines()
    .map { it.substringAfter("package:").trim() }  // 提取包名
    .filter { it.isNotEmpty() }  // 过滤空行
```

## 已知限制

1. **需要 Root 权限**：`su -c pm` 需要 Root 权限
   - ✅ PandaSU 本身就是 Root 管理工具，已经有 Root 权限
   
2. **性能**：执行 shell 命令比直接调用 API 稍慢
   - 但 492 个应用的处理时间仍然在可接受范围内（< 2 秒）

3. **兼容性**：某些设备的 `pm` 命令可能有不同的参数
   - 已添加异常处理，失败时回退到 PackageManager API
