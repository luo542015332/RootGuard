# PandaSU 环境安全评分问题解决报告

## 问题概述
**用户反馈**: "环境安全评分还是不对 我有装tricky store 还是扣分了 评分监测机制好像没有效果"

**核心问题**: 环境安全评分机制未能正确检测已安装的 Tricky Store 模块，导致错误扣分。

## 根本原因分析

### 1. Tricky Store 检测逻辑缺陷
原代码中Tricky Store检测只检查两个固定路径：
```kotlin
val hasTrickyStore = fileExists("/data/adb/tricky_store") || fileExists("/data/adb/modules/tricky_store")
```

**问题点**:
- 路径检查范围过窄
- 没有考虑模块安装路径的多样性
- 缺乏动态查找机制

### 2. 模块安装路径多样性
实际环境中，Tricky Store可能以多种路径安装：
1. `/data/adb/tricky_store` - 标准主目录
2. `/data/adb/modules/tricky_store` - 标准模块目录  
3. `/data/adb/modules/trickystore` - 无下划线版本
4. `/data/adb/tricky_store_current` - 新版本带后缀
5. `/data/adb/modules/ts` - 缩写版本
6. 其他包含"tricky"关键词的目录

### 3. 用户实际环境
从日志分析，设备环境为：
- Android HyperOS (MIUI 14/15)
- 使用 KernelSU Root方案
- 已安装 Tricky Store 模块
- 但PandaSU未能正确检测

## 解决方案

### 1. 增强路径检测
修改 `RootHider.kt` 中的Tricky Store检测逻辑：

```kotlin
// 增强的 Tricky Store 检测逻辑
val trickyStorePaths = listOf(
    "/data/adb/tricky_store",
    "/data/adb/modules/tricky_store",
    "/data/adb/modules/trickystore",  // 无下划线版本
    "/data/adb/tricky_store_current", // 新版本可能带后缀
    "/data/adb/modules/ts"           // 缩写版本
)

var hasTrickyStore = false
var trickyStorePath = ""
for (path in trickyStorePaths) {
    if (fileExists(path)) {
        hasTrickyStore = true
        trickyStorePath = path
        break
    }
}

// 额外检查：查找包含 "tricky" 的模块目录
if (!hasTrickyStore) {
    val trickyRelatedPaths = findDirectoriesContaining("/data/adb/modules", "tricky")
    if (trickyRelatedPaths.isNotEmpty()) {
        hasTrickyStore = true
        trickyStorePath = trickyRelatedPaths[0]
    }
}
```

### 2. 添加动态查找辅助函数
新增 `findDirectoriesContaining()` 函数，用于动态查找包含特定关键词的目录：

```kotlin
private suspend fun findDirectoriesContaining(basePath: String, keyword: String): List<String> = withContext(Dispatchers.IO) {
    val results = mutableListOf<String>()
    try {
        val baseDir = File(basePath)
        if (baseDir.exists() && baseDir.isDirectory) {
            baseDir.listFiles()?.forEach { dir ->
                if (dir.isDirectory && dir.name.contains(keyword, ignoreCase = true)) {
                    results.add(dir.absolutePath)
                }
            }
        }
    } catch (e: Exception) {
        Logger.e("findDirectoriesContaining failed for $basePath with keyword $keyword: ${e.message}")
    }
    return@withContext results
}
```

### 3. 改进日志记录
- 记录所有检查的路径
- 记录找到的模块路径（如果找到）
- 详细记录检测结果和扣分情况
