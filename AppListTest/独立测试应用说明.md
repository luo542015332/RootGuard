# 独立应用列表测试工具

## 问题说明
之前我创建的是从 PandaSU 项目复制代码的测试工具，不够独立。现在重新创建了一个**完全独立的测试应用**。

## 新的独立测试应用
**包名**: `com.applist.test`
**应用名**: 应用列表测试

## 测试方法
这个独立应用包含 3 种获取应用列表的方法：

### 1. PackageManager API (GET_META_DATA)
- 使用 `packageManager.getInstalledPackages(PackageManager.GET_META_DATA)`
- 这是最常用的 Android API 方法

### 2. pm list packages -3 命令
- 使用 Shell 命令 `pm list packages -3`
- 只获取第三方应用（排除系统应用）

### 3. PackageManager API (MATCH_ALL)
- 使用 `packageManager.getInstalledPackages(PackageManager.MATCH_ALL)`
- 尝试获取所有应用，包括隐藏的应用

## 测试步骤
1. 运行 `build.bat` 构建 APK
2. 安装到 Android 16 设备
3. 打开应用，依次点击 3 个按钮
4. 查看每种方法获取到的：
   - 总应用数
   - 用户应用数
   - 系统应用数
   - 关键应用（微信、QQ、王者荣耀、梦幻西游）是否找到
   - 前 30 个应用列表

## 关键差异说明
这个独立应用与 PandaSU 的区别：
1. **包名完全不同**：`com.applist.test` vs `com.rootguard.app`
2. **不需要 Root 权限**：只需普通应用权限
3. **功能单一**：只测试获取应用列表，不包含其他功能
4. **代码简洁**：没有 PandaSU 的复杂逻辑，专注测试
