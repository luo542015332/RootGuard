# v1.4.32 更新说明

## 问题分析

从 v1.4.30 日志发现：
- PackageManager 只返回 301 个应用
- 微信、QQ、英雄联盟不在列表中
- 但系统已验证：`pm list packages` 命令返回 492 个包，微信 QQ 都在其中

**根本原因：**
`PackageManager.getInstalledApplications()` 在某些 Android 版本或 ROM（特别是 MIUI）上有已知的限制，无法返回所有已安装应用。

## 修复方案

**强制使用 `pm list packages` 命令**获取完整的应用列表：
1. 先尝试执行 `su -c pm list packages` 命令
2. 如果成功（返回的包名列表不为空），使用这个列表
3. 逐个调用 `pm.getApplicationInfo()` 获取应用详细信息
4. 添加详细的调试日志，记录 pm 命令的退出码和错误输出

**为什么使用 pm 命令：**
- `pm list packages` 是系统命令，不受 PackageManager API 的限制
- 能获取所有已安装的包名（包括被 PackageManager API 遗漏的应用）
- 之前日志显示 pm 命令返回 492 个包，包含微信 QQ

**性能考虑：**
- 虽然需要逐个查询 ApplicationInfo（492 次），但：
  - 查询操作很快（<1ms/个）
  - 总时间约 0.5-1 秒
  - 图标加载已延迟到 UI 层，不影响加载时间
