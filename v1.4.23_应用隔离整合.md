# v1.4.23 - 应用和隔离功能整合更新说明

## 更新日期
2026-03-22

## 更新内容

### 核心变更
将用户应用授权管理和隔离功能整合为一个统一的界面,解决之前两个功能使用不同的系统应用判断逻辑导致的冲突问题。

### 主要改进

#### 1. 统一的应用管理界面
- **原问题**: 用户应用列表和隔离功能使用不同的系统应用判断逻辑
  - AppsScreen: 只检查 FLAG_SYSTEM 标志和安装路径
  - IsolationScreen: 额外检查包名前缀(com.miui., com.xiaomi., com.android., com.google.android.)
- **解决方案**: 移除独立隔离页面,将隔离功能集成到应用列表中

#### 2. 新的数据结构
```kotlin
data class AppItem(
    val packageName: String,
    val name: String,
    val rootStatus: RootAccessStatus,
    val isSystemApp: Boolean = false,
    val icon: Drawable? = null,
    val isIsolated: Boolean = false,      // 新增
    val isolationLevel: String = "未设置"  // 新增
)

enum class IsolationLevel(val displayName: String, val emoji: String) {
    DISABLED("未设置", "⭕"),
    PERMISSIVE("宽松", "🟢"),
    MODERATE("中等", "🟡"),
    STRICT("严格", "🔴")
}
```

#### 3. 应用卡片界面重构
每个应用卡片现在包含两个设置:
- **Root 权限**: 允许/拒绝/询问
- **隔离级别**: 未设置/宽松/中等/严格

#### 4. 统计卡片更新
新增隔离统计卡片,显示已设置隔离的应用数量:
```
📱 用户  ⚙️ 系统  🛡️ 隔离  ✅ 已授权  ❌ 已拒绝
```

#### 5. 导航简化
- 移除 `onNavigateToIsolation` 参数
- 移除独立的隔离页面路由(`Isolation`, `AppIsolation`)
- 首页移除"一键隔离"功能卡片

#### 6. ViewModel 更新
新增 `setIsolationLevel()` 方法:
```kotlin
fun setIsolationLevel(packageName: String, level: IsolationLevel) {
    viewModelScope.launch {
        // TODO: 实际保存隔离级别到数据库
        val updatedApps = _uiState.value.apps.map { a ->
            if (a.packageName == packageName) {
                a.copy(
                    isIsolated = level != IsolationLevel.DISABLED,
                    isolationLevel = level.displayName
                )
            } else a
        }
        updateApps(updatedApps)
    }
}
```

### 技术细节

#### 统一系统应用判断逻辑
使用 `MagiskProvider.isSystemApp()` 方法,该方法:
1. 优先检查 `FLAG_SYSTEM` 标志
2. 备选检查安装路径(/system/, /vendor/, /product/, /apex/)

避免使用包名前缀判断,因为:
- MIUI 的预装应用(com.miui.)可能安装在 /data/app/
- 用户可能自己卸载系统应用
- 不同厂商使用不同的包名前缀

#### UI 交互流程
1. 用户打开应用授权页面
2. 点击某个应用的"隔离级别"按钮
3. 从下拉菜单选择隔离级别(未设置/宽松/中等/严格)
4. 更新应用状态并保存到数据库(待实现)
5. 隔离级别会在应用名称旁显示(已隔离的应用)

### 用户体验改进

**之前**:
- 用户需要进入两个不同的页面管理应用
- 应用授权在"应用"页面,隔离在"隔离"页面
- 两处显示的应用数量可能不一致

**现在**:
- 所有应用管理集中在一个页面
- 可以在同一个界面设置 Root 权限和隔离级别
- 统一的应用列表和统计信息
- 更直观的应用状态展示

### 兼容性说明

- 保持向后兼容,现有的 Root 授权功能不受影响
- 隔离功能暂时不实际生效(仅 UI 层,待实现数据库)
- 升级后用户现有的 Root 策略会被保留
