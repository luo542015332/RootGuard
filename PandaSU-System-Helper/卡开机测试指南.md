# PandaSU System Helper - 卡开机测试指南

## ⚠️ 风险分析

### 低风险（✅ 安全）
- `customize.sh` - 标准安装脚本，无风险
- `service.sh` - 后台循环，sleep 60秒，不占用CPU
- `post-fs-data.sh` - 只执行resetprop，风险极低
- `system.prop` - 18个标准属性，都是常见属性

### 中风险（⚠️ 注意）
- `sepolicy.rule` - **SELinux规则可能冲突**，已移除（安全版本）

## 🛡️ 安全测试版本

已生成**无sepolicy.rule**的安全版本：
- 位置：`C:\Users\Administrator\Desktop\PandaSU-System-Helper\dist\PandaSU-System-Helper-v1.0.0-SAFE.zip`
- 大小：约5KB
- **不会卡开机**（已移除唯一风险点）

## 🧪 安全测试步骤

### 阶段1：手动测试（不刷模块）

**操作**：双击运行 `手动测试脚本.bat`

**测试内容**：
1. 创建配置目录
2. 测试DenyList写入
3. 测试属性伪装
4. 验证属性值
5. 测试TrickyStore同步

**判断标准**：
- ✅ 所有命令返回成功
- ✅ 属性值正确显示
- **预计耗时**：30秒
- **风险**：零风险

### 阶段2：模块安装测试

**前提**：阶段1全部成功

**操作**：
1. 把 `PandaSU-System-Helper-v1.0.0-SAFE.zip` 传到手机
2. 打开Magisk/KernelSU → 模块 → 从本地安装
3. **不要重启**，先检查模块是否激活

**检查命令**：
```bash
adb shell "ls -l /data/adb/modules/PandaSU-System-Helper"
adb shell "cat /data/adb/pandasu/config"
adb shell "cat /data/adb/pandasu/module.log"
```

**判断标准**：
- ✅ 模块目录存在
- ✅ 配置文件存在
- ✅ 日志文件有内容
- **预计耗时**：2分钟
- **风险**：极低（不重启）

### 阶段3：重启测试

**前提**：阶段2成功

**操作**：
1. **确保能进Recovery/TWRP**（重要！）
2. 重启手机
3. 观察开机时间

**判断标准**：
- ✅ 开机时间正常（< 2分钟）
- ✅ 没有卡在启动画面
- ✅ 能正常解锁进入系统
- **预计耗时**：3-5分钟
- **风险**：低（有Recovery备份）

### 阶段4：功能验证

**操作**：
1. 打开PandaSU应用
2. 查看环境检测评分
3. 对比安装模块前后的分数

**预期结果**：
- 评分从55分提升到75-85分
- 模块状态显示"已安装"
- TrickyStore白名单自动同步

**查看模块日志**：
```bash
adb shell "cat /data/adb/pandasu/module.log"
```

## 🆘 卡开机应对方案

### 情况1：能进Recovery/TWRP
**操作**：
1. 进入Recovery
2. 找到模块目录：`/data/adb/modules/`
3. 删除PandaSU-System-Helper文件夹
4. 重启

**命令**：
```bash
adb shell "rm -rf /data/adb/modules/PandaSU-System-Helper"
```

### 情况2：不能进Recovery
**操作**：
1. 开机时按住音量下（或上）进入Fastboot
2. 连接电脑
3. 刷入原厂boot.img
4. 重启后模块自动失效

### 情况3：完全无法开机
**操作**：
1. 使用厂商工具线刷完整固件
2. 或联系售后恢复
