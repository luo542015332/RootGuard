# PandaSU v1.4.20 更新说明

## 修复内容
尝试使用组合 flags 获取所有已安装应用，包括禁用的和已卸载的应用。

## 问题描述
### v1.4.19 的问题
虽然使用了 `MATCH_ALL` flag，但 PackageManager 仍然只返回 301 个应用。
用户应用仍然只显示 4 个，微信、QQ 等关键应用不在列表中。

## 新的修复方案

### Android 13+ (API 33)
组合多个 flags 以确保获取所有应用：
```kotlin
val flags = PackageManager.MATCH_ALL.toLong() or
           PackageManager.MATCH_DISABLED_COMPONENTS.toLong() or
           PackageManager.MATCH_DISABLED_UNTIL_USED_COMPONENTS.toLong() or
           PackageManager.MATCH_UNINSTALLED_PACKAGES.toLong()
pm.getInstalledApplications(
    PackageManager.ApplicationInfoFlags.of(flags)
)
```

### Android 12 及以下
同样组合多个 flags：
```kotlin
val flags = PackageManager.GET_META_DATA or
            PackageManager.GET_DISABLED_COMPONENTS or
            PackageManager.GET_UNINSTALLED_PACKAGES
pm.getInstalledApplications(flags)
```

## Flag 说明

| Flag | 作用 |
|-------|------|
| `MATCH_ALL` | 获取所有应用（所有用户、所有 profile） |
| `MATCH_DISABLED_COMPONENTS` | 包含禁用的组件 |
| `MATCH_DISABLED_UNTIL_USED_COMPONENTS` | 包含禁用直到使用的组件 |
| `MATCH_UNINSTALLED_PACKAGES` | 包含已卸载的应用（数据保留） |
| `GET_META_DATA` | 获取应用的元数据 |

## 可能的备选方案

如果 v1.4.20 仍然失败，可能需要考虑：

### 方案 A: 使用 ADB 命令直接获取
```kotlin
val process = Runtime.getRuntime().exec(arrayOf("su", "-c", "pm", "list", "packages"))
val output = process.inputStream.bufferedReader().readText()
val packages = output.lines()
    .map { it.substringAfter("package:") }
    .map { packageName -> pm.getApplicationInfo(packageName, 0) }
```

### 方案 B: 检查权限问题
可能需要额外的权限或 Manifest 中添加：
```xml
<uses-permission android:name="android.permission.QUERY_ALL_PACKAGES" />
```

### 方案 C: 使用用户 ID
尝试获取不同用户 ID 下的应用：
```kotlin
for (userId in userManager.userProfiles) {
    pm.getInstalledApplicationsAsUser(flags, userId.id)
}
```
