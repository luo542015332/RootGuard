# 日志功能闪退修复报告

## 问题描述
提取手机日志时发生闪退

## 问题分析

### 1. LogsViewModel 缺少错误处理
**文件**: `ui/screens/logs/LogsViewModel.kt`

**问题**:
- 第 45-47 行：当发生异常时，只更新 `isLoading = false`
- 没有记录错误日志
- UI 可能处于空状态，导致后续访问空列表时崩溃

### 2. MagiskProvider.getLogs() 执行可能失败的命令
**文件**: `data/magisk/MagiskProvider.kt`

**问题**:
- 第 325 行：执行 `magisk -l` 命令
- 如果 Magisk 未安装或版本不支持，会抛出异常
- 日志解析逻辑不够健壮，可能无法处理异常格式的日志

### 3. 日志解析缺少异常保护
**问题**:
- `parseLogLine()` 方法没有 try-catch 保护
- 解析失败时可能导致崩溃

---

## 修复方案

### 1. 改进 LogsViewModel 错误处理 ✅

**修改内容**:
- 添加 `Logger` 导入和必要的导入
- 在异常处理中添加错误日志记录
- 发生错误时显示一条错误日志条目，而不是空列表

**修复代码**:
```kotlin
catch (e: Exception) {
    Logger.e("Failed to load logs", e)
    // 发生错误时显示一条错误日志
    val errorLog = LogEntry(
        id = "error_${System.currentTimeMillis()}",
        message = "无法加载日志: ${e.message}",
        level = LogLevel.ERROR,
        timestamp = SimpleDateFormat("MM-dd HH:mm:ss", Locale.getDefault()).format(Date())
    )
    _uiState.update {
        it.copy(
            logs = listOf(errorLog),
            isLoading = false
        )
    }
}
```

### 2. 改进 MagiskProvider.getLogs() ✅

**修改内容**:
- 在执行 logcat 命令前检查 Magisk 是否安装
- 使用 `logcat` 替代 `magisk -l`（更可靠）
- 限制日志数量为 100 条，避免内存问题
- 改进错误处理和默认日志

**修复代码**:
```kotlin
try {
    // 检查 Magisk 是否安装
    if (!isMagiskInstalled()) {
        throw Exception("Magisk 未安装")
    }

    // 执行 logcat 命令获取系统日志（更可靠的方法）
    val process = Runtime.getRuntime().exec(
        arrayOf("su", "-c", "logcat", "-d", "-v", "time", "*:I")
    )
    val output = process.inputStream.bufferedReader().readText()
    process.waitFor()

    // 解析最近的日志
    val lines = output.lines().takeLast(100) // 只取最后 100 条
    lines.forEachIndexed { index, line ->
        if (line.isNotBlank() && !line.contains("Magisk")) {
            logs.add(parseLogLine(line, index))
        }
    }
} catch (e: Exception) {
    Logger.e("Failed to get logs", e)
    // 返回初始化日志而不是空列表
    logs.add(MagiskLog(
        id = "0",
        message = "无法获取日志: ${e.message}",
        level = LogLevel.WARNING,
        timestamp = dateFormat.format(Date())
    ))
}
```

### 3. 改进 parseLogLine() 方法 ✅

**修改内容**:
- 添加 try-catch 保护解析逻辑
- 改进日志消息提取逻辑
- 解析失败时创建默认日志而不是崩溃
- 限制消息长度为 500 字符

**修复代码**:
```kotlin
private fun parseLogLine(line: String, id: Int): MagiskLog {
    return try {
        // ... 日志解析逻辑
    } catch (e: Exception) {
        // 如果解析失败，创建一个默认日志
        MagiskLog(
            id = id.toString(),
            message = line.take(500), // 限制长度避免过长
            level = LogLevel.INFO,
            timestamp = dateFormat.format(Date())
        )
    }
}
```
