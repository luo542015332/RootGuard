# PandaSU 一键隔离功能诊断报告

## 问题描述
用户报告：一键隔离没起作用，点哪个应用都是一样的效果

## 检查时间
2026-03-24 01:18

## 核心文件检查
- ✅ `OneClickIsolationHelper.kt` - 存在
- ✅ `IsolationViewModel.kt` - 存在  
- ✅ `RootHider.kt` - 存在

## 调用链分析

### 1. 用户点击应用
```
用户点击应用 -> toggleAppIsolation() 被调用
```

### 2. toggleAppIsolation() 逻辑
```kotlin
fun toggleAppIsolation(appInfo: OneClickIsolationHelper.AppInfo) {
    viewModelScope.launch {
        try {
            val existing = isolationDataStore.getConfigForAppSync(appInfo.packageName)
            if (existing != null && existing.isEnabled) {
                // 已有配置且启用 -> 移除隔离
                rootHider.removeIsolation(appInfo.packageName)
                isolationDataStore.saveIsolationConfig(existing.copy(isEnabled = false))
                _uiState.update { it.copy(successMessage = "已取消 ${appInfo.appName} 的隔离") }
            } else {
                // 无配置或未启用 -> 应用隔离
                val config = oneClickIsolationHelper.generateIsolationConfig(appInfo, _uiState.value.selectedOneClickPreset)
                isolationDataStore.saveIsolationConfig(config)
                rootHider.applyIsolation(config)
                _uiState.update { it.copy(successMessage = "已为 ${appInfo.appName} 启用隔离") }
            }
            clearMessageAfterDelay()
        } catch (e: Exception) {
            _uiState.update { it.copy(errorMessage = "操作失败: ${e.message}") }; clearMessageAfterDelay()
        }
    }
}
```

### 3. 生成配置
```
generateIsolationConfig() -> 根据应用类别和预设生成隔离配置
```

### 4. 应用隔离 (applyIsolation)
```
applyIsolation() 执行7个步骤：
1. configureMagiskHide() - 添加应用到 Magisk/KernelSU 隐藏列表
2. hideSuForPackage() - 隐藏 su 二进制文件
3. hideMagiskFilesForPackage() - 隐藏 Magisk 文件
4. hideBusyboxForPackage() - 隐藏 Busybox
5. hideXposedForPackage() - 隐藏 Xposed
6. applyCustomProps() - 应用自定义系统属性
7. disableRootForPackage() - 禁用 Root 访问
```

## 可能的失败点

### 🔴 高优先级问题
1. **Root 权限不足** - 导致所有 su 命令失败
2. **Magisk/KernelSU 未安装** - configureMagiskHide 失败
3. **配置保存失败** - isolationDataStore 保存配置失败

### 🟡 中优先级问题
4. **挂载点权限问题** - hideSuForPackage 等挂载操作失败
5. **UI 状态更新问题** - _uiState.update 未正确更新
6. **应用信息获取失败** - getApplicationInfo 异常

### 🟢 低优先级问题
7. **沙箱规则生成失败** - generateIsolationConfig 异常

## 问题分析

### 最可能的原因：Root 权限问题
如果 PandaSU 没有 Root 权限，所有 su 命令都会静默失败，导致：
- configureMagiskHide() 失败
- hideSuForPackage() 失败
- 所有需要 su 权限的操作都失败

### 次要原因：配置保存问题
如果 isolationDataStore 保存配置失败，会导致：
- 每次点击都认为应用无配置
- 每次都生成新配置
- 状态无法正确切换

### UI 反馈问题
如果 UI 状态更新失败，用户看不到操作结果，感觉"没起作用"

## 修复建议

## 结论

一键隔离功能的核心逻辑是完整的，但可能因以下原因失效：

1. **Root 权限问题** - 最可能的原因，需要优先检查
2. **配置保存问题** - 需要验证数据存储机制
3. **错误处理不足** - 需要添加更详细的日志和错误提示

建议先修复日志记录，然后根据日志输出定位具体问题。
