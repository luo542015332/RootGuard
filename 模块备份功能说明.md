# 模块备份和恢复功能

## 功能概述

为 RootGuard (PandaSU) 应用添加了完整的模块备份和恢复功能，允许用户：

1. **备份已安装的模块** - 将模块文件和配置保存到本地存储
2. **恢复已备份的模块** - 从备份中恢复模块到系统
3. **查看备份状态** - 在模块列表中显示是否有备份及备份时间
4. **删除备份** - 管理备份文件

---

## 技术实现

### 1. 模块备份管理器 (ModuleBackupManager)

**文件**: `app/src/main/java/com/rootguard/app/data/repository/ModuleBackupManager.kt`

**主要功能**:
- `backupModule(moduleId: String)` - 备份指定模块
- `restoreModule(backupId: String)` - 恢复指定备份
- `getAllBackups()` - 获取所有备份列表
- `deleteBackup(backupId: String)` - 删除备份
- `hasBackup(moduleId: String)` - 检查模块是否有备份
- `getLatestBackup(moduleId: String)` - 获取模块的最新备份

**实现细节**:
- 备份文件保存在: `/sdcard/Android/data/com.rootguard.app/files/module_backups/`
- 每个备份包含一个 ZIP 文件和备份信息 JSON
- 使用 `java.util.zip` 进行文件压缩和解压
- 所有操作使用协程在 IO 线程执行

**备份流程**:
1. 从 `/data/adb/modules/{moduleId}` 读取模块文件
2. 压缩整个模块目录到 ZIP 文件
3. 创建备份信息 JSON（包含模块名称、版本、作者等）
4. 保存到应用外部存储目录

**恢复流程**:
1. 读取备份信息
2. 解压 ZIP 文件到临时目录
3. 复制到 `/data/adb/modules/{moduleId}`
4. 清理临时文件

### 3. 模块卡片 UI 更新

**文件**: `app/src/main/java/com/rootguard/app/ui/screens/modules/ModulesScreen.kt`

**ModuleItem 数据类扩展**:
```kotlin
data class ModuleItem(
    val id: String,
    val name: String,
    val version: String,
    val description: String,
    val author: String,
    val isEnabled: Boolean,
    val hasBackup: Boolean = false,          // 新增：是否有备份
    val backupTime: Long? = null            // 新增：备份时间

    fun getBackupTime(): String {
        return backupTime?.let {

        } ?: "无备份"
    }
}
```

## 用户使用流程

### 备份模块
1. 打开"模块管理"页面
2. 找到想要备份的模块
3. 点击模块卡片上的"备份"按钮
4. 等待备份完成
5. 备份成功后会显示提示对话框

### 恢复模块
1. 打开"模块管理"页面
2. 找到已备份的模块（会有备份图标和时间显示）
3. 点击"恢复"按钮
4. 等待恢复完成
5. 恢复成功后建议重启设备

### 备份状态识别
- **无备份** - 显示"备份"按钮
- **有备份** - 显示备份图标 📦 和备份时间，显示"恢复"按钮

## 数据存储结构

### 备份目录结构
```
/sdcard/Android/data/com.rootguard.app/files/module_backups/
├── {backup_id}/
│   ├── {backup_id}.zip          # 模块文件压缩包
│   └── backup_info.json         # 备份信息
└── ...
```

### 备份信息 JSON 格式
```json
{
  "id": "module_id_1711000000000",
  "moduleId": "module_name",
  "name": "Module Display Name",
  "version": "1.0.0",
  "description": "Module Description",
  "author": "Author Name",
  "backupTime": 1711000000000,
  "backupPath": "/path/to/backup.zip",
  "fileSize": 123456
}
```

## 权限要求

备份和恢复功能需要以下权限：
- ✅ Root 权限 - 访问 `/data/adb/modules` 目录
- ✅ 存储权限 - 应用外部存储（Android 11+ 不需要用户授权）

---

## 已知限制

1. **Root 权限** - 必须在已 Root 的设备上使用
2. **Magisk/KernelSU** - 需要安装 Magisk 或 KernelSU
3. **存储空间** - 备份文件会占用设备存储空间
4. **恢复冲突** - 恢复模块时会覆盖现有文件
