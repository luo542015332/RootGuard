# 前台服务崩溃修复报告

## 问题描述

从崩溃日志中发现两个主要问题：

### 问题1：RootRequestMonitorService 前台服务崩溃
**错误信息**：
```
ForegroundServiceDidNotStartInTimeException: Context.startForegroundService() did not then call Service.startForeground()
ServiceRecord{... com.rootguard.app/.service.RootRequestMonitorService}
```

**发生位置**：
```
at com.rootguard.app.receiver.BootReceiver.onReceive(BootReceiver.kt:20)
at android.app.ContextImpl.startForegroundService(ContextImpl.java:2020)
```

**根因**：
- BootReceiver 通过 `startForegroundService()` 启动 RootRequestMonitorService
- Android 系统要求前台服务必须在 5 秒内调用 `startForeground()`
- RootRequestMonitorService 的 `onCreate()` 方法没有调用 `startForeground()`

### 问题2：PandaSUApp 类找不到
**错误信息**：
```
ClassNotFoundException: Didn't find class "com.rootguard.app.PandaSUApp"
```

**可能原因**：
- 这是一个旧的日志，可能与之前的编译或安装问题有关
- 当前代码中 PandaSUApp.kt 文件存在且正确

```kotlin
private fun createNotificationChannel() {
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
        val channel = NotificationChannel(
            CHANNEL_ID,
            "Root 监控服务",
            NotificationManager.IMPORTANCE_LOW
        ).apply {
            description = "监控 Root 权限请求"
        }

        val notificationManager = getSystemService(Context.NOTIFICATION_SERVICE) as NotificationManager
        notificationManager.createNotificationChannel(channel)
    }
}

private fun createNotification(): Notification {
    val pendingIntent = android.app.PendingIntent.getActivity(
        this,
        0,
        Intent(this, MainActivity::class.java),
        android.app.PendingIntent.FLAG_IMMUTABLE
    )

    return NotificationCompat.Builder(this, CHANNEL_ID)
        .setContentTitle("RootGuard")
        .setContentText("正在监控 Root 权限请求")
        .setSmallIcon(R.drawable.ic_notification)
        .setContentIntent(pendingIntent)
        .setOngoing(true)
        .build()
}
```

## 技术背景

### 前台服务要求

从 Android 8.0 (API 26) 开始，系统对前台服务有严格要求：

1. **必须通过 startForegroundService() 启动**
   - 不能直接使用 startService()
   - 必须声明前台服务类型（`foregroundServiceType`）

2. **必须在 5 秒内调用 startForeground()**
   - 从服务创建到调用 startForeground() 的时间不能超过 5 秒
   - 否则会抛出 `ForegroundServiceDidNotStartInTimeException`

3. **必须显示通知**
   - 前台服务必须显示持续的通知
   - 通知必须包含标题和内容

4. **必须创建通知渠道（Android O+）**
   - 通知渠道必须在显示通知前创建
   - 导渠 ID 必须与构建通知时使用的 ID 一致

### BootReceiver 中的启动流程

```kotlin
class BootReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context, intent: Intent) {
        if (intent.action == Intent.ACTION_BOOT_COMPLETED) {
            // 启动 Root 监控服务
            val serviceIntent = Intent(context, RootMonitorService::class.java)
            context.startForegroundService(serviceIntent)  // ✓ 正确：onStartCommand 中调用了 startForeground()

            // 启动 Root 请求监控服务
            val monitorIntent = Intent(context, RootRequestMonitorService::class.java).apply {
                action = RootRequestMonitorService.ACTION_START_MONITORING
            }
            context.startForegroundService(monitorIntent)  // ✗ 错误：onCreate() 中未调用 startForeground()
        }
    }
}
```

## 对比分析

### RootMonitorService（正确）

```kotlin
override fun onCreate() {
    super.onCreate()
    createNotificationChannel()  // 创建通知渠道
}

override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
    val notification = createNotification()
    startForeground(NOTIFICATION_ID, notification)  // ✓ 在 onStartCommand 中调用
    return START_STICKY
}
```

**问题**：虽然正确，但 `onCreate()` 中只创建了通知渠道，没有调用 `startForeground()`。
**风险**：如果 `onStartCommand()` 延迟执行，可能超过 5 秒限制。

**建议**：最佳实践是在 `onCreate()` 中调用 `startForeground()`，以确保服务创建后立即成为前台服务。

### RootRequestMonitorService（已修复）

修复前：
```kotlin
override fun onCreate() {
    super.onCreate()
    Logger.d("RootRequestMonitorService created")  // ✗ 没有调用 startForeground()
}
```

修复后：
```kotlin
override fun onCreate() {
    super.onCreate()
    createNotificationChannel()  // 创建通知渠道
    startForeground(NOTIFICATION_ID, createNotification())  // ✓ 立即成为前台服务
    Logger.d("RootRequestMonitorService created")
}
```

1. **通知权限**：确保应用已授予 POST_NOTIFICATIONS 权限（Android 13+）
2. **前台服务类型**：AndroidManifest.xml 中已正确声明 `foregroundServiceType="specialUse"`
3. **通知图标**：确保 `ic_notification` 资源存在（已存在于项目中）
4. **通知渠道**：通知渠道 ID 不能与其他服务冲突（已区分：RootMonitorService 使用 CHANNEL_ID="rootguard_monitor"，RootRequestService 使用 CHANNEL_ID="rootguard_requests"）

通过在 `RootRequestMonitorService.onCreate()` 中添加 `startForeground()` 调用，修复了前台服务崩溃问题。这符合 Android 系统对前台服务的严格要求，确保服务在 5 秒内成为前台服务并显示通知。

修复后的代码遵循了 Android 最佳实践，提高了应用的稳定性和可靠性。
