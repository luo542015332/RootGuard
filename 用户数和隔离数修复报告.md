### 1. 改进系统应用判断 ✅

**文件**: `app/src/main/java/com/rootguard/app/data/magisk/MagiskProvider.kt`

**问题描述**:
- 原有代码只依赖 `FLAG_SYSTEM` 标志来判断系统应用
- 在某些 ROM（如 MIUI）中，用户应用可能被错误地标记为系统应用
- 导致用户数显示不准确

**修复方案**:
添加了 `isSystemApp()` 方法，采用多层判断逻辑：

```kotlin
private fun isSystemApp(appInfo: android.content.pm.ApplicationInfo): Boolean {
    // 1. 检查 FLAG_SYSTEM 标志
    val hasSystemFlag = (appInfo.flags and android.content.pm.ApplicationInfo.FLAG_SYSTEM) != 0
    
    if (!hasSystemFlag) {
        return false
    }
    
    // 2. 进一步检查安装路径
    // 用户应用通常安装在 /data/app/ 下
    // 系统应用安装在 /system/、/vendor/、/product/ 等目录下
    val sourceDir = appInfo.sourceDir ?: return true
    
    val isUserInstalled = sourceDir.contains("/data/app/") || 
                         sourceDir.contains("/data/data/")
    
    return !isUserInstalled
}
```

**改进效果**:
- ✅ 准确识别用户应用和系统应用
- ✅ 修复被错误标记为系统应用的用户应用
- ✅ 适应不同 ROM 的特性（MIUI、ColorOS 等）

**测试建议**:
1. 检查应用列表中的用户数是否与实际一致
2. 验证从应用商店安装的应用是否被正确归类
3. 对比 ADB 命令结果：
   ```bash
   adb shell pm list packages -3 | wc -l  # 应该与用户数一致
   ```

### 2. 添加详细日志 ✅

**文件**: 
- `app/src/main/java/com/rootguard/app/data/magisk/MagiskProvider.kt`
- `app/src/main/java/com/rootguard/app/ui/screens/apps/AppsViewModel.kt`

**问题描述**:
- 原有代码缺少详细的调试日志
- 应用数量显示不对时，无法快速定位问题
- 难以追踪应用加载过程

**修复方案**:

#### 2.1 MagiskProvider.kt 添加的日志

**已有日志**（保留）:
```kotlin
Logger.d("PackageManager returned ${packages.size} packages")
Logger.d("Added ${apps.size} apps to list: $userApps user apps, $systemApps system apps")
apps.filter { !it.isSystemApp }.take(10).forEach {
    Logger.d("User app: ${it.packageName} - ${it.appName}")
}
```

#### 2.2 AppsViewModel.kt 新增日志

```kotlin
private fun updateApps(apps: List<AppItem>) {
    val userApps = apps.count { !it.isSystemApp }
    val systemApps = apps.count { it.isSystemApp }
    val granted = apps.count { it.rootStatus == RootAccessStatus.GRANTED }
    val denied = apps.count { it.rootStatus == RootAccessStatus.DENIED }
    val prompt = apps.count { it.rootStatus == RootAccessStatus.PROMPT }

    // 记录应用数量统计
    Logger.d("Apps loaded: Total=${apps.size}, User=$userApps, System=$systemApps")
    Logger.d("Root status: Granted=$granted, Denied=$denied, Prompt=$prompt")

    _uiState.update { state -> ... }
}
```

**日志输出示例**:
```
D/PandaSU: PackageManager returned 492 packages
D/PandaSU: Added 492 apps to list: 133 user apps, 359 system apps
D/PandaSU: User app: com.example.app1 - Example App 1
D/PandaSU: User app: com.example.app2 - Example App 2
...
D/PandaSU: Apps loaded: Total=492, User=133, System=359
D/PandaSU: Root status: Granted=5, Denied=3, Prompt=125
```

**改进效果**:
- ✅ 可以追踪应用加载的完整过程
- ✅ 快速诊断应用数量不对的问题
- ✅ 便于后续调试和问题排查

**使用方法**:
提取日志后搜索关键词：
```bash
adb logcat -d | grep "Apps loaded"
adb logcat -d | grep "User app:"
adb logcat -d | grep "PackageManager returned"
```
