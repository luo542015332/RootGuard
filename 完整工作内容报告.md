- **修复**:
  - 添加多个 KernelSU 管理器支持（Weishu 和 Tiann）

  - 检查数据库表结构并动态选择查询

### v1.4.2 → v1.4.3 (versionCode 16→17)

- **改进**: 优化数据库查询逻辑
- **状态**: BUILD SUCCESSFUL

### v1.4.3 → v1.4.4 (versionCode 17→18)
- **修复**: 用户授权列表应用状态全部错误
- **根本原因**:
  - KernelSU 的 policy 值：0=拒绝, 1=允许
  - Magisk 的 policy 值：0=询问, 1=拒绝, 2=允许
  - 代码使用 Magisk 的值映射逻辑，导致 KernelSU 环境下状态全部错误
- **修复**:
  - 添加 `getCurrentRootType()` public 方法
  - 根据 Root 类型动态选择 policy 值映射逻辑
  - 改进 `getAppsWithRootStatus()` 和 `setAppRootAccess()` 方法
- **状态**: BUILD SUCCESSFUL

### v1.4.4 → v1.4.5 (versionCode 18→19)
- **修复**: KernelSU 管理器检测逻辑严重 bug
- **根本原因**:
  - `ksudbWeishu = context.packageManager.getPackageInfo(KERNELSU_PACKAGE_WEISHU, 0)` 没有包装在 try-catch
  - 当 Weishu KernelSU 未安装时，getPackageInfo() 抛出异常
  - `ksudbWeishu != null` 判断无效，因为异常已经抛出
  - 导致整个方法失败，无法获取授权信息
- **修复**:
  - 将 Weishu KernelSU 的检测也包装在 try-catch 中
  - 使用 `when` 表达式替代 `if-else` 链
  - 同时修复 `getAppPolicies()` 和 `setAppPolicy()` 中的相同问题
- **状态**: BUILD SUCCESSFUL

### v1.4.5 → v1.4.6 (versionCode 19→20)

## 四、修复的主要问题

### 1. 前台服务崩溃
- **问题**: `ForegroundServiceDidNotStartInTimeException`
- **根因**: BootReceiver 启动 RootRequestMonitorService 后，服务未在 5 秒内调用 `startForeground()`
- **修复**: 在 `onCreate()` 中添加 `startForeground()`，创建通知渠道和通知
- **文件**: `RootRequestMonitorService.kt`

### 2. 模块备份/恢复失败
- **问题**: Java 的 `File.exists()` 和 `File.copyRecursively()` 无法访问需要 Root 权限的目录
- **修复**: 使用 `su -c zip` 和 `su -c cp` 命令
- **文件**: `ModuleBackupManager.kt`

### 3. 用户应用数量显示不全
- **问题**: 只显示 5 个用户应用，实际有 133 个
- **原因**: 获取应用图标或名称失败时，整个应用被跳过
- **修复**: 改进错误处理，使用默认图标
- **文件**: `MagiskProvider.kt`

### 4. 系统应用判断错误
- **问题**: MIUI 等自定义 ROM 将某些用户应用标记为系统应用
- **修复**: 检查安装路径，排除 `/data/app/` 下的应用
- **文件**: `MagiskProvider.kt` 的 `isSystemApp()` 方法

### 5. 模块列表显示非模块文件
- **问题**: 显示 README.md 等文件
- **修复**: 改用 `find -type d` 只列出目录
- **文件**: `MagiskProvider.kt`

### 6. KernelSU 兼容性
- **问题**: 代码只支持 Magisk，不支持 KernelSU
- **修复**:
  - 添加 Root 类型自动检测
  - 支持 Weishu 和 Tiann KernelSU 管理器
  - 实现不同的数据库查询逻辑
  - 实现 KernelSU denylist 命令
- **文件**: `MagiskProvider.kt`, `RootHider.kt`

### 7. 用户授权列表显示问题（持续修复中）
- **问题 1**: 显示数量不正确（36个）
  - 原因: 查询错误的数据库路径
  - 修复: 添加 KernelSU 数据库路径检测
- **问题 2**: 应用状态全部错误
  - 原因: 使用 Magisk 的 policy 值映射
  - 修复: 根据 Root 类型动态映射
- **问题 3**: 检测逻辑 bug
  - 原因: getPackageInfo() 未包装在 try-catch
  - 修复: 添加异常处理
- **当前状态**: v1.4.6 添加详细日志，等待日志分析
