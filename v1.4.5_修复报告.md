# v1.4.5 修复报告

## 问题描述

用户报告 v1.4.4 版本仍然存在相同的问题：用户授权列表显示不正确（仍然是 36 个应用，且状态错误）。

## 根本原因分析

经过代码审查，发现 `MagiskProvider.kt` 中的 KernelSU 管理器检测逻辑存在严重 bug：

### 问题代码（v1.4.4）

```kotlin
val ksudbWeishu = context.packageManager.getPackageInfo(KERNELSU_PACKAGE_WEISHU, 0)
val ksudbTiann = try {
    context.packageManager.getPackageInfo(KERNELSU_PACKAGE_TIANN, 0)
    true
} catch (e: Exception) {
    false
}

if (ksudbWeishu != null) {
    // 使用 Weishu KernelSU
}
```

### 问题分析

1. `getPackageInfo()` 方法会抛出异常如果包未安装
2. `ksudbWeishu` 变量要么是 `PackageInfo` 对象，要么抛出异常
3. 当 Weishu KernelSU 未安装时，代码会直接抛出异常而不是返回 false
4. 这导致整个策略查询失败，无法获取任何授权信息

## 修复方案

将 `ksudbWeishu` 的检测也包装在 try-catch 块中：

```kotlin
val ksudbWeishu = try {
    context.packageManager.getPackageInfo(KERNELSU_PACKAGE_WEISHU, 0)
    true
} catch (e: Exception) {
    false
}
val ksudbTiann = try {
    context.packageManager.getPackageInfo(KERNELSU_PACKAGE_TIANN, 0)
    true
} catch (e: Exception) {
    false
}

when {
    ksudbWeishu -> {
        Logger.d("Using Weishu KernelSU manager: $KERNELSU_DB_WEISHU")
        KERNELSU_DB_WEISHU
    }
    ksudbTiann -> {
        Logger.d("Using Tiann KernelSU manager: $KERNELSU_DB_TIANN")
        KERNELSU_DB_TIANN
    }
    else -> {
        Logger.w("No KernelSU manager found, defaulting to Weishu")
        KERNELSU_DB_WEISHU
    }
}
```

## 修复的文件

1. **MagiskProvider.kt**
   - 修复 `getAppPolicies()` 方法中的 KernelSU 管理器检测逻辑
   - 修复 `setAppPolicy()` 方法中的相同问题
   - 使用 `when` 表达式替代 `if-else` 链，代码更清晰
