# 模块安装问题诊断指南

## 问题现象
点击"安装模块"按钮后,文件选择器没有弹出,或者选择文件后没有反应。

## 可能原因

### 1. 文件选择器未启动
**症状**: 点击"安装模块"按钮后没有任何反应

**检查方法**:
```bash
# 查看 logcat 中的错误
adb logcat -d | grep -i "install\|picker\|uri"
```

**解决方案**:
- 检查权限: 文件选择器需要 READ_EXTERNAL_STORAGE 权限
- 检查 AndroidManifest.xml 中是否正确声明了权限

### 2. 文件复制失败
**症状**: 选择文件后立即报错"无法读取文件"

**检查方法**:
- 查看日志中的 `installModuleFromUri called with URI` 行
- 检查 URI 是否有效
- 检查临时文件是否创建成功

**解决方案**:
- 检查 AndroidManifest.xml 中是否有 `<queries>` 标签允许访问文件管理器
- 检查文件是否损坏

### 3. 模块验证失败
**症状**: 显示"无效的 Magisk 模块文件"

**检查方法**:
```bash
# 手动检查 zip 文件
unzip -l module.zip | grep module.prop
```

**解决方案**:
- 确保模块文件包含 `module.prop` 文件
- 确保模块文件格式正确

### 4. Root 权限不足
**症状**: 显示"模块安装失败"但没有详细错误信息

**检查方法**:
```bash
# 检查 su 命令是否可用
adb shell su -c id

# 检查 magisk/ksu 命令
adb shell su -c "which magisk || which ksu"
```

**解决方案**:
- 确保设备已正确 root
- 确保应用有 root 权限
- 检查 Root 类型 (Magisk/KernelSU)

### 5. KernelSU 兼容性问题
**症状**: 使用 KernelSU 时安装失败

**检查方法**:
```bash
# 检查 KernelSU 是否安装
adb shell ls /data/adb/ksu

# 检查 KSU 版本
adb shell su -c ksu -v
```

**解决方案**:
- 当前代码已支持 KernelSU
- 确保 KernelSU 版本兼容
- 检查日志中的 `Detected root type` 信息

## 调试步骤

### 1. 清空 logcat
```bash
adb logcat -c
```

### 2. 启动应用并尝试安装模块
- 打开 PandaSU 应用
- 进入模块管理页面
- 点击"安装模块"按钮
- 选择一个有效的 Magisk 模块 zip 文件

### 3. 提取日志
```bash
# 提取所有 PandaSU 日志
adb logcat -d -v time | grep "PandaSU" > pandasu_install.log

# 提取模块安装相关日志
adb logcat -d -v time | grep -i "install\|module" > module_install.log

# 提取完整的 logcat
adb logcat -d -v time > full_logcat.log
```

### 4. 查看日志关键信息
查找以下关键词:
- `installModuleFromUri called with URI` - 确认文件选择器启动
- `File copied to` - 确认文件复制成功
- `Module validation passed` - 确认模块验证通过
- `Detected root type` - 确认 Root 类型检测
- `Install result` - 确认安装结果

## 常见错误信息

### "无法读取文件"
- 原因: ContentResolver 无法访问 URI
- 解决: 检查文件权限和 URI 格式

### "无效的 Magisk 模块文件"
- 原因: zip 文件中没有 module.prop
- 解决: 使用正确的模块文件

### "模块安装失败"
- 原因: Root 权限不足或命令执行失败
- 解决: 检查 Root 权限和日志中的详细错误

### "安装失败: ..."
- 原因: 各种异常情况
- 解决: 查看异常堆栈跟踪
