## 修复的问题

### 1. 应用列表显示问题 ✅
**问题描述**: 用户应用只显示 5 个,但实际有 36 个

**原因**: `AppsScreen.kt` 中的 `currentFilter` 本地状态没有与 `viewModel.uiState.filter` 同步

**修复内容**:
```kotlin
// AppsScreen.kt
var currentFilter by remember { mutableStateOf(uiState.filter) }

// 当 viewModel 的 filter 状态变化时,更新本地状态
LaunchedEffect(uiState.filter) {
    currentFilter = uiState.filter
}
```

**提交**: 2afb954

**验证**: 
- 日志显示: `Added 301 apps to list: 36 user apps, 265 system apps`
- 统计卡片应显示正确的数量

### 2. 模块安装失败问题 ✅
**问题描述**: 模块安装功能没有反应,安装失败

**原因**: 
- 只支持 Magisk 的 `magisk --install-module` 命令
- KernelSU 不支持此命令
- 没有自动检测 Root 类型

**修复内容**:
- 添加 `detectRootType()` 方法自动检测 KernelSU/Magisk
- 添加 `installKernelSUModule()` 方法支持 KernelSU 模块安装
  - 解压 zip 文件到临时目录
  - 检查 module.prop 验证模块
  - 复制文件到 `/data/adb/modules/{moduleId}/`
  - 设置权限和启用状态
- 添加 `installMagiskModule()` 方法改进 Magisk 安装
- 添加错误日志和退出码检查

**提交**: 5c95a16

**验证**: 新 APK 已安装,等待用户测试

## 技术细节

### Root 类型检测
```kotlin
private suspend fun detectRootType(): String = withContext(Dispatchers.IO) {
    // 1. 检查 KernelSU 特征文件并验证 ksu 命令
    KERNELSU_PATHS.forEach { path ->
        if (File(path).exists()) {
            val process = Runtime.getRuntime().exec("su -c ksu -v")
            if (process.waitFor() == 0) {
                return@withContext ROOT_TYPE_KERNELSU
            }
        }
    }

    // 2. 检查 Magisk 特征文件并验证 magisk 命令
    val process = Runtime.getRuntime().exec("su -c magisk -v")
    if (process.waitFor() == 0) {
        return@withContext ROOT_TYPE_MAGISK
    }

    // 3. 默认使用 Magisk 兼容模式
    return@withContext ROOT_TYPE_MAGISK
}
```

### KernelSU 模块安装流程
1. 解压 zip 文件到 `/data/local/tmp/module_{timestamp}/`
2. 检查 `module.prop` 验证模块
3. 从 `module.prop` 读取模块 ID
4. 创建 `/data/adb/modules/{moduleId}/` 目录
5. 复制所有文件到模块目录
6. 设置权限 `chmod -R 755`
7. 移除 `disable` 文件启用模块
