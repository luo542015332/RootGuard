## 二、工作内容

### 1. 主应用合并双注入模式（核心功能）

**目标**: 在一个 APK 中同时支持 Zygisk 和 LSPosed 两种注入方式

**实现步骤**:
1. 创建 `app/src/main/assets/zygisk_module/` 目录
2. 复制 Zygisk 模块文件（module.prop, customize.sh, .so 库）
3. 编写 `ZygiskModuleInstaller.kt` - 自动安装模块到 Magisk
4. 修改 `PandaSUApp.kt` - 应用启动时自动检测并安装模块

### 2. 增强 Zygisk 模块（v3.1.0）

**新增隐藏配置**:
- 扩展 `HIDE_NAMES[]` - 添加更多检测工具关键词
- 扩展 `HIDE_PATHS[]` - 添加更多隐藏路径

**新增隐藏项**:
- Root 工具: ksu, kernelsu, magiskhide, shamiko
- Xposed 变体: LSPosed, Riru, EdXposed, APatch, KPwn
- 检测工具: 爱玩机, Scene, 核心破解, AWZ
- Manager: MagiskManager, KernelSU Manager

### 3. Android 16 兼容性适配

**问题**: LSPosed 1.9.2 无法在 Android 16 上注入 system_server

**解决方案**:
- Zygisk 模块作为主要隐藏手段（系统级）
- LSPosed 模块作为辅助（App 层过滤）
- 两者互补，增强隐藏效果

## 四、存在的问题

### ❌ 已知限制

1. **Android 16 上 LSPosed 无法注入系统进程**
   - 原因: LSPosed 1.9.2 与 Android 16 不兼容
   - 影响: PM Command / PM Conventional APIs 检测无法隐藏
   - 解决: 使用 Magisk Denylist 或 Shamiko 模块

2. **Libc/Syscall File Detection 无法完全隐藏**
   - 原因: 需要内核层 Hook，当前仅文件系统层
   - 解决: 需要集成完整 inline hook 库

3. **Xposed Modules 列表检测**
   - 原因: 需要 Hook LSPosed 内部 API
   - 解决: 在 Magisk Denylist 中勾选检测应用

### ⚠️ 临时解决方案

在手机上手动操作:
1. 打开 **Magisk → 设置 → Magisk Denylist**
2. 勾选需要隐藏的应用:
   - 爱玩机
   - Scene
   - 核心破解
   - LSPosed Manager
   - AppList Detector
3. 安装 **Shamiko** 模块（可选，进一步隐藏）
