# 版本号同步修复方案

## 🔍 问题分析

### 当前状态

**app/build.gradle**:
```gradle
versionCode 2
versionName "1.1.0"
```

**GitHub Actions (build.yml)**:
```yaml
APK 文件名: PandaSU-v1.1.${{ github.run_number }}.apk
Release 标签: v1.1.${{ github.run_number }}
```

### 问题

1. **版本号不同步**
   - APK 内部版本固定为 1.1.0
   - 文件名使用 run_number（如 v1.1.123）
   - 容易混淆

2. **versionCode 没有递增**
   - 固定为 2，不利于版本管理

3. **release 标签混乱**
   - 每次构建都会生成不同的 release 标签
   - 但实际版本没变

---

## 🎯 修复方案

### 方案 1：使用固定版本（推荐用于稳定版）

**优点**:
- 版本号稳定，清晰
- 适合正式发布

**缺点**:
- 需要手动更新版本号

**操作步骤**:

1. **修改 build.gradle**（固定版本号）:
```gradle
versionCode 3
versionName "1.2.0"  // 手动递增主版本号
```

2. **修改 build.yml**（使用固定版本）:
```yaml
# 使用 build.gradle 中的版本号
- name: Get version from build.gradle
  id: get_version
  run: |
    VERSION=$(grep versionName app/build.gradle | awk '{print $2}' | tr -d '"')
    echo "version=$VERSION" >> $GITHUB_OUTPUT

- name: Rename APK
  run: mv app/build/outputs/apk/debug/app-debug.apk app/build/outputs/apk/debug/PandaSU-${{ steps.get_version.outputs.version }}.apk

- name: Create Release
  if: github.event.inputs.create_release == 'true'
  uses: softprops/action-gh-release@v1
  with:
    tag_name: v${{ steps.get_version.outputs.version }}
    name: PandaSU v${{ steps.get_version.outputs.version }}
```

---

### 方案 2：自动递增版本号（推荐用于开发版）

**优点**:
- 自动管理版本号
- 每次构建自动递增

**缺点**:
- 版本号增长快

**操作步骤**:

1. **修改 build.gradle**（支持版本变量）:
```gradle
def versionMajor = 1
def versionMinor = 2
def versionPatch = System.getenv("BUILD_NUMBER")?.toInteger() ?: 0

android {
    defaultConfig {
        versionCode versionMajor * 10000 + versionMinor * 100 + versionPatch
        versionName "${versionMajor}.${versionMinor}.${versionPatch}"
    }
}
```

2. **修改 build.yml**（传递构建编号）:
```yaml
- name: Build Debug APK
  run: BUILD_NUMBER=${{ github.run_number }} ./gradlew assembleDebug --no-daemon
  env:
    BUILD_NUMBER: ${{ github.run_number }}

- name: Rename APK
  run: mv app/build/outputs/apk/debug/app-debug.apk app/build/outputs/apk/debug/PandaSU-${{ env.VERSION_NAME }}.apk
```

---

### 方案 3：混合方案（推荐）

**主版本**: 手动管理（1.2.0）
**构建号**: 自动递增（用于区分测试版）

**操作**:

1. **修改 build.gradle**:
```gradle
versionCode System.getenv("BUILD_NUMBER")?.toInteger() ?: 2
versionName "1.2.0"  // 主版本号手动管理
```

2. **修改 build.yml**:
```yaml
# Release 时使用主版本号
- name: Create Release
  if: github.event.inputs.create_release == 'true'
  uses: softprops/action-gh-release@v1
  with:
    tag_name: v1.2.0  # 使用固定主版本
    name: PandaSU v1.2.0
    body: |
      ## PandaSU v1.2.0
      Build #${{ github.run_number }}
```

## 📊 版本号规范建议

**版本格式**: 主版本.次版本.修订号
- **主版本** (1.x.x): 重大更新，不兼容旧版本
- **次版本** (x.2.x): 新功能，向后兼容
- **修订号** (x.x.0): Bug 修复，小改动

**versionCode**: 单调递增的整数
- 每次发布必须增加
- Google Play 要求唯一性
