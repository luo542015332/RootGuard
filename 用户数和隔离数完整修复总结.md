## 🔍 问题分析

### 问题 1: 用户数显示不正确

**可能原因**:
1. 应用加载时出错（某些应用处理失败被跳过）
2. 系统应用判断错误（某些用户应用被错误标记为系统应用）
3. 数据未刷新（应用列表没有及时更新）
4. 缺少日志（无法确定具体原因）

**实际数据验证** (通过 ADB):
```bash
$ adb shell pm list packages | wc -l
492  # 总应用数

$ adb shell pm list packages -3 | wc -l
133  # 用户应用

$ adb shell pm list packages -s | wc -l
359  # 系统应用
```

### 问题 2: 隔离数始终为 0

**根因**: 
- 用户的设备使用的是 **KernelSU** 而不是 Magisk
- `RootHider.kt` 只支持 Magisk 的 `magisk --denylist` 命令
- 在 KernelSU 环境下，Magisk 命令失败，导致：
  - 隔离配置无法应用到系统
  - `getIsolatedApps()` 返回空列表
  - 隔离数显示为 0

**验证**:
```bash
# Magisk 命令（失败）
$ su -c magisk --denylist list
magisk: command not found

# KernelSU 命令（成功）
$ ksu denylist list
com.example.app
com.another.app
...
```

## ✅ 解决方案

### 方案 1: 改进系统应用判断

**文件**: `MagiskProvider.kt`

**改进内容**:
- 添加 `isSystemApp()` 方法
- 采用多层判断逻辑：
  1. 检查 `FLAG_SYSTEM` 标志
  2. 检查安装路径（`/data/app/` 为用户应用，`/system/` 等为系统应用）
- 解决了 MIUI 等 ROM 将用户应用错误标记为系统应用的问题

**代码实现**:
```kotlin
private fun isSystemApp(appInfo: android.content.pm.ApplicationInfo): Boolean {
    // 检查 FLAG_SYSTEM 标志
    val hasSystemFlag = (appInfo.flags and android.content.pm.ApplicationInfo.FLAG_SYSTEM) != 0
    
    if (!hasSystemFlag) {
        return false
    }
    
    // 如果有 FLAG_SYSTEM 标志，进一步检查安装路径
    val sourceDir = appInfo.sourceDir ?: return true
    
    val isUserInstalled = sourceDir.contains("/data/app/") || 
                         sourceDir.contains("/data/data/")
    
    return !isUserInstalled
}
```

**效果**:
- ✅ 更准确地识别用户应用和系统应用
- ✅ 修复被错误标记的用户应用
- ✅ 适应不同 ROM 的特性

### 方案 3: 添加 KernelSU 支持

**文件**: `RootHider.kt`

**关键改进**:

#### 3.1 添加 Root 类型检测

```kotlin
private suspend fun detectRootType(): String = withContext(Dispatchers.IO) {
    cachedRootType?.let { return@withContext it }
    
    // 检查 KernelSU
    val hasKernelSU = KERNELSU_PATHS.any { path -> File(path).exists() }
    
    if (hasKernelSU) {
        val process = Runtime.getRuntime().exec("su -c ksu --version")
        process.waitFor()
        if (process.exitValue() == 0) {
            cachedRootType = ROOT_TYPE_KERNELSU
            return@withContext ROOT_TYPE_KERNELSU
        }
    }
    
    // 检查 Magisk
    val hasMagisk = MAGISK_PATHS.any { path -> File(path).exists() }
    
    if (hasMagisk) {
        val process = Runtime.getRuntime().exec("su -c magisk -c")
        process.waitFor()
        if (process.exitValue() == 0) {
            cachedRootType = ROOT_TYPE_MAGISK
            return@withContext ROOT_TYPE_MAGISK
        }
    }
    
    // 默认使用 Magisk
    cachedRootType = ROOT_TYPE_MAGISK
    return@withContext ROOT_TYPE_MAGISK
}
```

#### 3.2 修改隔离配置方法

```kotlin
private suspend fun configureMagiskHide(config: IsolationConfig): Boolean = withContext(Dispatchers.IO) {
    val rootType = detectRootType()
    
    when (rootType) {
        ROOT_TYPE_KERNELSU -> {
            // 使用 KernelSU 命令
            Runtime.getRuntime().exec("su -c ksu denylist add ${config.packageName}")
        }
        ROOT_TYPE_MAGISK -> {
            // 使用 Magisk 命令
            Runtime.getRuntime().exec("su -c magisk --denylist add ${config.packageName}")
        }
    }
}
```

#### 3.3 修改其他相关方法

- `removeIsolation()` - 支持两种 Root 方案的移除操作
- `getIsolatedApps()` - 从不同的数据源读取隔离列表
- `checkHideStatus()` - 检查不同的隔离状态

**效果**:
- ✅ 自动检测 KernelSU 或 Magisk
- ✅ 在 KernelSU 环境下正常工作
- ✅ 隔离数正确显示
- ✅ 保持向后兼容（默认使用 Magisk 兼容模式）

## 📝 注意事项

1. **Root 类型缓存**: `cachedRootType` 变量会在首次检测后缓存结果，如果设备 Root 方案发生变化，需要重启应用

2. **兼容模式**: 如果无法检测 Root 类型，默认使用 Magisk 兼容模式，确保基本功能可用

3. **日志级别**: 新增的日志使用 Debug 级别，正式版本可以考虑降低日志级别

4. **性能优化**: Root 类型检测结果会缓存，避免重复检测

5. **异常处理**: 所有关键操作都有异常处理，失败时会记录日志并尝试备用方法
