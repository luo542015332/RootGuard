# PandaSU v1.4.19 更新说明

## 修复内容
修复了 Android 13+ 设备上无法获取完整应用列表的关键问题。

## 问题描述
### 用户反馈
- PackageManager 只返回 301 个应用
- 用户应用只有 4 个：PandaSU、美团、小米换机、英雄联盟手游
- **微信、QQ、英雄联盟手游（王者荣耀）等关键应用完全不显示**
- 系统应用 297 个，但大多数应该是用户应用被错误归类

### 日志分析
```
PackageManager returned 301 packages
Key apps found in list: []  ← 微信、QQ 完全不在列表中
Apps loaded: Total=301, User=4, System=297
```

## 根本原因
在 Android 13+ (API 33) 上：
- **错误做法**: 使用 `ApplicationInfoFlags.of(0)` 来获取应用列表
- **问题**: `ApplicationInfoFlags.of(0)` 不等同于旧版 API 的 `0`，无法获取所有应用
- **正确做法**: 必须使用 `ApplicationInfoFlags.of(PackageManager.MATCH_ALL.toLong())`

在 Android 12 及以下：
- **错误做法**: 使用 `getInstalledApplications(0)`
- **正确做法**: 使用 `getInstalledApplications(PackageManager.GET_META_DATA)`

## 修复方案
### Android 13+ (API 33)
```kotlin
pm.getInstalledApplications(
    PackageManager.ApplicationInfoFlags.of(
        PackageManager.MATCH_ALL.toLong()
    )
)
```

### Android 12 及以下
```kotlin
@Suppress("DEPRECATION")
pm.getInstalledApplications(PackageManager.GET_META_DATA)
```

## 技术细节
### MATCH_ALL Flag 的作用
`PackageManager.MATCH_ALL` 是 Android 13+ 引入的新 flag，用于获取：
- 所有用户安装的应用
- 所有系统应用
- 所有禁用的应用
- 所有工作资料中的应用
- 所有其他 profile 的应用

如果不使用 `MATCH_ALL`，可能只能获取当前用户的部分应用。

### GET_META_DATA Flag
`GET_META_DATA` 包含：
- `MATCH_UNINSTALLED_PACKAGES` 的功能
- 所有应用的元数据信息

## 已知限制
- Android 13+ 使用新的 `ApplicationInfoFlags` API
- Android 12 及以下使用旧版 int flags
- 不同 Android 版本的行为可能略有差异
