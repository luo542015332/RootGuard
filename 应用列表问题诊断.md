# PandaSU 应用列表问题诊断指南

## 问题现象
用户应用只显示 5 个,但实际有 36 个用户应用

## 日志分析
从最新日志可以看到:
- `Added 301 apps to list: 36 user apps, 265 system apps`
- `Apps loaded: Total=301, User=36, System=265`

这说明后端数据获取是正常的,问题可能在前端显示或过滤器设置

## 已修复的问题

### 1. 过滤器状态同步问题 (AppsScreen.kt)
**问题**: `currentFilter` 本地状态没有与 `viewModel.uiState.filter` 同步

**修复**: 添加 `LaunchedEffect` 来同步状态
```kotlin
var currentFilter by remember { mutableStateOf(uiState.filter) }

// 当 viewModel 的 filter 状态变化时,更新本地状态
LaunchedEffect(uiState.filter) {
    currentFilter = uiState.filter
}
```

## 用户需要检查的事项

### 1. 检查当前过滤器设置
- 打开应用后,查看顶部是否有"筛选: xxx"的提示
- 如果有,点击"清除筛选"按钮
- 或者点击筛选按钮,选择"全部应用"

### 2. 检查统计卡片
查看应用列表上方的统计卡片,应该显示:
- 📱 用户: 36
- ⚙️ 系统: 265
- ✅ 已授权: 0
- ❌ 已拒绝: 0

如果"用户"显示的不是 36,说明统计有问题

### 3. 尝试切换筛选
1. 点击右上角的筛选按钮
2. 选择"📱 用户应用" → 应该显示 36 个用户应用
3. 选择"⚙️ 系统应用" → 应该显示 265 个系统应用
4. 选择"全部应用" → 应该显示 301 个应用

### 4. 检查是否有搜索关键词
查看搜索框是否输入了内容,如果有,清除搜索框

### 5. 清除应用数据并重新安装
如果以上方法都不行,尝试:
1. 设置 → 应用 → PandaSU → 清除数据
2. 重新打开应用

## 技术细节

### 应用检测逻辑
应用通过以下方式判断系统应用:
```kotlin
// 1. 检查 FLAG_SYSTEM 标志
val hasSystemFlag = (appInfo.flags and ApplicationInfo.FLAG_SYSTEM) != 0

// 2. 检查安装路径
// 用户应用: /data/app/ 或 /data/data/
// 系统应用: /system/, /vendor/, /product/
val isUserInstalled = sourceDir.contains("/data/app/") || 
                     sourceDir.contains("/data/data/")
return hasSystemFlag && !isUserInstalled
```

### 已知的 MIUI 特殊情况
MIUI ROM 可能把某些用户应用标记为 FLAG_SYSTEM,
但它们的安装路径仍在 /data/app/ 下,
这种情况下会被正确识别为用户应用。
