# PandaSU v1.2.0 启动闪退问题分析报告

## 问题概述

**现象**：应用启动后立即闪退
**错误**：`ClassNotFoundException: Didn't find class "com.rootguard.app.PandaSUApp"`

## 崩溃日志分析

```
java.lang.RuntimeException: Unable to instantiate application com.rootguard.app.PandaSUApp
Caused by: java.lang.ClassNotFoundException: Didn't find class "com.rootguard.app.PandaSUApp" on path: DexPathList[[zip file "base.apk"],nativeLibraryDirectories=[...]]
```

**关键信息**：
- 错误类型：`ClassNotFoundException`
- 缺失类：`com.rootguard.app.PandaSUApp`（应用主类）
- APK 文件：存在（15MB），但内容损坏/不完整

## 根本原因

### 1. 项目目录结构混乱

项目根目录存在两套代码结构：
- `app/` 目录：**空的**（之前没有源代码）
- `RootGuard/app/` 目录：**包含完整源代码**（55 个 Kotlin 文件）

### 2. GitHub Actions 构建问题

- GitHub Actions 配置在项目根目录运行
- 根目录的 `app/` 目录之前是空的
- 导致构建的 APK 不包含任何编译代码
- 虽然有资源文件（图标等），但没有 DEX 文件

### 3. 版本标记误导

- APK 的 `AndroidManifest.xml` 显示 versionCode=3, versionName=1.2.0
- 但这个版本号的 APK 实际上是"空壳"
- 误导性：看起来是正确版本，实际内容是损坏的

## 技术细节

### APK 结构验证

正确的 APK 应该包含：
- `AndroidManifest.xml`
- `classes.dex`（编译的 Kotlin/Java 代码）
- `resources.arsc`
- `res/`（资源文件）
- `lib/`（native 库，如果有）

损坏的 APK 包含：
- `AndroidManifest.xml`
- `res/`（图标等资源）
- ❌ **缺少** `classes.dex`
- ❌ **缺少** 编译的代码

### 为什么会闪退

1. **应用启动**：
   - Android 系统读取 `AndroidManifest.xml`
   - 查找 `android:name=".PandaSUApp"`
   - 尝试加载 `com.rootguard.app.PandaSUApp` 类

2. **类加载失败**：
   - 系统在 DEX 文件中查找类
   - DEX 文件不存在或为空
   - 抛出 `ClassNotFoundException`

3. **崩溃**：
   - 无法创建 Application 实例
   - 进程终止，应用闪退
