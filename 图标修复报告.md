# GitHub Actions 图标修复报告

## 问题描述

GitHub Actions 构建失败，错误信息：

```
ERROR: resource mipmap/ic_launcher (aka com.rootguard.app:mipmap/ic_launcher) not found.
ERROR: resource mipmap/ic_launcher_round (aka com.rootguard.app:mipmap/ic_launcher_round) not found.
```

## 根本原因

所有 `mipmap-*/ic_launcher*.xml` 文件只包含注释，没有实际引用任何资源：

```xml
<?xml version="1.0" encoding="utf-8"?>
<!-- 图标文件位于: ic_launcher.png -->
<!-- 此文件仅用于兼容，实际使用 PNG -->
```

Android 资源系统需要实际的资源引用（`<bitmap>`、`<vector>` 或 PNG 文件），不能只有注释。

## 解决方案

使用 Android 系统默认图标作为临时方案，确保 GitHub Actions 可以正常构建。

### 修改的文件

所有分辨率文件夹中的图标配置文件（共 12 个）：

1. `mipmap-hdpi/ic_launcher.xml` (48x48)
2. `mipmap-hdpi/ic_launcher_round.xml`
3. `mipmap-mdpi/ic_launcher.xml` (36x36)
4. `mipmap-mdpi/ic_launcher_round.xml`
5. `mipmap-xhdpi/ic_launcher.xml` (72x72)
6. `mipmap-xhdpi/ic_launcher_round.xml`
7. `mipmap-xxhdpi/ic_launcher.xml` (96x96)
8. `mipmap-xxhdpi/ic_launcher_round.xml`
9. `mipmap-xxxhdpi/ic_launcher.xml` (144x144)
10. `mipmap-xxxhdpi/ic_launcher_round.xml`
11. `mipmap-anydpi-v26/ic_launcher.xml`
12. `mipmap-anydpi-v26/ic_launcher_round.xml`

### 新的配置格式

```xml
<?xml version="1.0" encoding="utf-8"?>
<!-- PNG 图标文件 - 96x96 -->
<bitmap xmlns:android="http://schemas.android.com/apk/res/android"
    android:src="@android:drawable/ic_menu_gallery"
    android:tint="#FF6200" />
```

使用 Android 自带的 `ic_menu_gallery` 图标，并添加橙色（#FF6200）色调。

## 后续步骤

### 替换为熊猫图标（用户提供的图片）

如果用户想要使用熊猫图标，需要：

1. 准备以下分辨率的 PNG 图片：
   - 36x36 (mdpi)
   - 48x48 (hdpi)
   - 72x72 (xhdpi)
   - 96x96 (xxhdpi)
   - 144x144 (xxxhdpi)

2. 将图片重命名为 `ic_launcher.png` 和 `ic_launcher_round.png`

3. 放置到对应的 `mipmap-{resolution}/` 文件夹

4. 删除 XML 配置文件，或修改为 PNG 引用方式

### 使用图标处理工具

可以使用以下工具快速生成所需分辨率的图标：

- **在线工具**: appicon.co (免费)
- **ImageMagick**: 自动处理脚本 `update-icon.ps1`
- **Android Studio**: Image Asset Studio

## 注意事项

- 当前使用的是系统默认图标，不影响应用功能
- 橙色色调与 RootGuard 主题色一致
- 如需自定义图标，请准备各分辨率的 PNG 文件并替换
- 建议使用正方形、高对比度的图标，确保在设备上清晰可见
