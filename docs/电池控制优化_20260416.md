# 电池控制功能优化报告

**时间**: 2026-04-16

---

## 问题分析

原代码存在以下问题：

### 1. 静默失败
所有命令都加了 `2>/dev/null || true`，无论写入成功还是失败，exit code 都是 0。

### 2. 无验证机制
写入 sysfs 节点后不回读验证。

### 3. 主线程阻塞
`isSuAvailable()` 在主线程同步执行。

### 4. 代码冗余
路径散落各处，`BatteryUtil` 和 `ThermalUtil` 职责重叠。

---

## 修改内容

### BatteryUtil.kt

1. 新增 `CtrlResult` 结果类（含成功数/失败数）
2. 统一节点路径常量
3. 所有控制方法异步执行
4. 写入后回读验证
5. 旧 API 标记 `@Deprecated`

### BatteryInfoActivity.kt

1. 简化 UI 布局
2. 显示具体修改节点数

### ThermalUtil.kt

1. 去除静默失败
2. 异步执行框架
3. 写入验证

---

## 新 API

| 方法 | 说明 |
|------|------|
| `spoofBatteryTemp(tempC)` | 伪造电池温度 |
| `setChargeCurrent(mA)` | 设置充电电流 |
| `enableFastCharge()` | 开启快充 |

温控相关统一用 `ThermalUtil.disableThermalThrottle()`。
