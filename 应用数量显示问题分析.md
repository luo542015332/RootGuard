# 应用数量显示问题分析与解决方案

## 问题描述

用户反馈：用户数和隔离数显示不正确。

## 代码架构分析

### 数据流

```
AppsScreen
  ↓
AppsViewModel.loadApps()
  ↓
GetAppsUseCase()
  ↓
MagiskRepository.getAppsWithRootStatus()
  ↓
MagiskProvider.getAllInstalledApps()
  ↓
PackageManager.getInstalledApplications()
```

### 显示位置

在 `AppsScreen.kt` 的统计卡片中显示：
- 📱 用户应用数：`uiState.userAppCount`
- ⚙️ 系统应用数：`uiState.systemAppCount`
- ✅ 已授权数：`uiState.grantedCount`
- ❌ 已拒绝数：`uiState.deniedCount`

### 计算逻辑

在 `AppsViewModel.kt` 的 `updateApps()` 方法中：
```kotlin
val userApps = apps.count { !it.isSystemApp }
val systemApps = apps.count { it.isSystemApp }
val granted = apps.count { it.rootStatus == RootAccessStatus.GRANTED }
val denied = apps.count { it.rootStatus == RootAccessStatus.DENIED }
val prompt = apps.count { it.rootStatus == RootAccessStatus.PROMPT }
```

## 可能的原因

### 原因1：应用加载时出错（最可能）

**现象**：PackageManager 返回了应用列表，但在逐个处理时某些应用失败

**日志特征**：
```
Failed to process app[123]: com.example.app
java.lang.SecurityException: Permission denial
```

**代码位置**：`MagiskProvider.kt` 第 286-288 行
```kotlin
} catch (e: Exception) {
    Logger.e("Failed to process app[${index}]: ${appInfo.packageName}", e)
}
```

**影响**：应用加载失败时，该应用不会被添加到列表，导致总数减少。

### 原因2：图标加载失败

**现象**：应用信息获取成功，但图标加载失败

**日志特征**：
```
Failed to load icon for com.example.app: SecurityException
```

**代码位置**：`MagiskProvider.kt` 第 271-276 行
```kotlin
val icon = try {
    pm.getApplicationIcon(appInfo)
} catch (e: Exception) {
    Logger.w("Failed to load icon for $packageName: ${e.message}")
    pm.getDefaultActivityIcon()  // 使用默认图标
}
```

**影响**：代码已经处理了图标加载失败，使用默认图标，不应该影响应用数量。

### 原因3：应用名称获取失败

**现象**：图标加载成功，但应用名称获取失败

**日志特征**：
```
Failed to get app name: SecurityException
```

**代码位置**：`MagiskProvider.kt` 第 264-269 行
```kotlin
val appName = try {
    pm.getApplicationLabel(appInfo).toString()
} catch (e: Exception) {
    packageName  // 使用包名作为替代
}
```

**影响**：代码已经处理了名称获取失败，使用包名作为替代，不应该影响应用数量。

### 原因4：系统应用判断错误

**现象**：某些用户应用被错误地标记为系统应用

**代码位置**：`MagiskProvider.kt` 第 264 行
```kotlin
val isSystemApp = (appInfo.flags and android.content.pm.ApplicationInfo.FLAG_SYSTEM) != 0
```

**可能原因**：
- Android 系统将某些第三方应用标记为系统应用
- 预装的应用被标记为系统应用
- 使用了不同的 ROM 或厂商定制系统

### 原因5：数据未刷新

**现象**：应用数量显示的是旧数据，没有更新

**代码位置**：`AppsViewModel.kt` 第 41-52 行
```kotlin
private fun loadApps() {
    viewModelScope.launch {
        _uiState.update { it.copy(isLoading = true) }
        try {
            val apps = getApps()
            updateApps(apps)
        } catch (e: Exception) {
            _uiState.update { it.copy(isLoading = false) }
        }
    }
}
```

**影响**：如果 `loadApps()` 没有被调用或调用失败，数据不会更新。

### 2. 查看关键日志

#### 应用数量统计
```bash
# 查看 PackageManager 返回的应用总数
grep "PackageManager returned" logs/PandaSU/app_count_diagnosis.log

# 查看实际添加到列表的应用数
grep "Added.*apps" logs/PandaSU/app_count_diagnosis.log
```

**预期输出**：
```
D/PandaSU: PackageManager returned 123 packages
D/PandaSU: Added 123 apps to list: 45 user apps, 78 system apps
```

**问题诊断**：
- 如果两个数字不一致，说明有应用加载失败
- 如果用户应用数明显少于预期，可能有很多用户应用被错误标记为系统应用

#### 应用加载错误
```bash
# 查看应用加载失败的日志
grep "Failed to process app" logs/PandaSU/app_errors_diagnosis.log
```

**预期输出**（正常情况下）：
```
（应该为空或只有很少的错误）
```

**问题诊断**：
- 如果有很多失败，说明某些应用无法处理
- 记录失败的应用包名，分析失败原因

#### 图标加载警告
```bash
# 查看图标加载失败的警告
grep "Failed to load icon" logs/PandaSU/app_errors_diagnosis.log
```

**预期输出**：
```
W/PandaSU: Failed to load icon for com.example.app: ...
```

**问题诊断**：
- 如果图标加载失败很多，但不影响数量（有默认图标）
- 如果同时影响应用列表，需要检查异常处理

### 3. 手动验证应用列表

```bash
# 获取系统报告的所有应用数量
adb shell pm list packages -3 | wc -l

# 获取用户应用数量（不包含系统应用）
adb shell pm list packages -3 | grep -v "^package:com.android\|^package:android\|^package:com.miui\|^package:com.qualcomm" | wc -l

# 或者使用更精确的方法
adb shell pm list packages -3 | grep -v "^package:com.android" | wc -l
```

## 解决方案

### 方案1：改进错误处理（推荐）

**问题**：应用加载失败时，应用被完全跳过，导致数量不准确

**修改文件**：`app/src/main/java/com/rootguard/app/data/magisk/MagiskProvider.kt`

**修改代码**：

```kotlin
suspend fun getAllInstalledApps(): List<InstalledAppInfo> = withContext(Dispatchers.IO) {
    val apps = mutableListOf<InstalledAppInfo>()
    val failedApps = mutableListOf<String>()

    try {
        val pm = context.packageManager
        val packages = pm.getInstalledApplications(0)

        Logger.d("PackageManager returned ${packages.size} packages")

        packages.forEachIndexed { index, appInfo ->
            try {
                val packageName = appInfo.packageName
                val isSystemApp = (appInfo.flags and android.content.pm.ApplicationInfo.FLAG_SYSTEM) != 0
                val appName = try {
                    pm.getApplicationLabel(appInfo).toString()
                } catch (e: Exception) {
                    packageName
                }

                val icon = try {
                    pm.getApplicationIcon(appInfo)
                } catch (e: Exception) {
                    Logger.w("Failed to load icon for $packageName: ${e.message}")
                    pm.getDefaultActivityIcon()
                }

                apps.add(
                    InstalledAppInfo(
                        packageName = packageName,
                        appName = appName,
                        isSystemApp = isSystemApp,
                        icon = icon
                    )
                )
            } catch (e: Exception) {
                // 记录失败的应用
                val packageName = appInfo.packageName
                failedApps.add(packageName)
                Logger.e("Failed to process app[${index}]: $packageName - ${e.message}", e)
                // 尝试添加基本信息（至少包名）
                try {
                    val isSystemApp = (appInfo.flags and android.content.pm.ApplicationInfo.FLAG_SYSTEM) != 0
                    apps.add(
                        InstalledAppInfo(
                            packageName = packageName,
                            appName = packageName,
                            isSystemApp = isSystemApp,
                            icon = pm.getDefaultActivityIcon()
                        )
                    )
                } catch (e2: Exception) {
                    Logger.e("Failed to add minimal info for $packageName", e2)
                }
            }
        }

        val userApps = apps.count { !it.isSystemApp }
        val systemApps = apps.count { it.isSystemApp }
        Logger.d("Added ${apps.size} apps to list: $userApps user apps, $systemApps system apps")

        // 如果有失败的应用，打印警告
        if (failedApps.isNotEmpty()) {
            Logger.w("Failed to process ${failedApps.size} apps: ${failedApps.take(5)}...")
        }

        apps.filter { !it.isSystemApp }.take(10).forEach {
            Logger.d("User app: ${it.packageName} - ${it.appName}")
        }
    } catch (e: Exception) {
        Logger.e("Failed to get installed apps", e)
    }

    apps.sortedBy { it.appName }
}
```

**改进点**：
1. 记录所有失败的应用
2. 在应用处理失败时，尝试添加基本信息（包名、系统应用标记）
3. 打印失败应用的警告日志
4. 提高应用的完整性

### 方案2：改进系统应用判断

**问题**：某些用户应用被错误地标记为系统应用

**修改文件**：`app/src/main/java/com/rootguard/app/data/magisk/MagiskProvider.kt`

**修改代码**：

```kotlin
// 改进系统应用判断逻辑
private fun isSystemApp(appInfo: ApplicationInfo): Boolean {
    // 方法1：检查 FLAG_SYSTEM 标志
    if ((appInfo.flags and ApplicationInfo.FLAG_SYSTEM) != 0) {
        // 方法2：进一步检查是否是真正的系统应用
        // 如果是预装应用但可以卸载，通常不是系统应用
        return appInfo.sourceDir == null || appInfo.sourceDir.startsWith("/system/")
    }
    return false
}

// 在 getAllInstalledApps() 中使用
val isSystemApp = isSystemApp(appInfo)
```

**改进点**：
1. 不仅仅依赖 FLAG_SYSTEM 标志
2. 检查应用的安装路径
3. 更准确地判断系统应用和用户应用

## 常见问题解答

### Q1：为什么显示的应用数量比实际少？

A：可能有以下原因：
1. 某些应用加载失败（查看日志确认）
2. 用户应用被错误标记为系统应用
3. 数据没有及时刷新（尝试刷新）

### Q2：如何确认实际的应用数量？

A：使用以下命令：
```bash
adb shell pm list packages -3 | wc -l
```

### Q3：隔离数在哪里显示？

A：隔离数在"一键隔离"功能中显示，路径是：首页 → 一键隔离

查看 `IsolationViewModel.kt` 和 `IsolationScreen.kt` 了解隔离功能。

## 总结

应用数量显示不正确的问题可能有多种原因：
1. 应用加载失败（最常见）
2. 系统应用判断不准确
3. 数据没有刷新

建议优先使用诊断工具提取日志，然后根据日志内容选择相应的解决方案。
